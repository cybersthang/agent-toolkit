# Spec: `/adopt-pattern` scaffold

## Problem

Adopting one pattern from an upstream repo (mattpocock / spec-kit / ECC /
karpathy) currently takes **5-7 manual touchpoints**:

1. Write hook code in `templates/claude/hooks/<name>.py`
2. Write slash command markdown in `templates/claude/commands/<name>.md`
3. Wire into `templates/claude/settings.json` hook chain
4. Write test file in `tests/test_<name>.py`
5. Add attribution block to `NOTICE`
6. Update `templates/CLAUDE.md` hook table + slash command list
7. (optional) Add memory snippet to `templates/memory/`

Session 2026-05-24 measured: ~ 250 LOC across 5-7 files for a single
adoption (git_guardrails + /constitution). Friction makes the toolkit
score **6/10 on Extensibility** — the lowest of all 6 scoring
dimensions, beaten by mattpocock (8), ECC (9), spec-kit (7), karpathy (7).

## Goal

A slash command `/adopt-pattern <upstream-url>` that:

1. Reads the upstream `SKILL.md` (or equivalent) at the URL
2. Asks 3-5 GRILL questions ABOUT the pattern (Why fit toolkit? Which
   enforcement tier? Stack-agnostic or framework-specific overlay?)
3. Stubs the 7 touchpoint files with `_TODO` placeholders filled in
4. Surfaces a `tasks.md` listing exact LOC to write per file
5. STOPS for DEV review before any actual code is written

Anti-goal: an "automatic adoption" that pulls upstream code wholesale.
Adoption REQUIRES adapting to toolkit conventions (`_common.py`,
`get_enforce_mode`, hook envelope format) — full automation would
re-introduce the lock-in we just removed in v0.13.

## Acceptance evals

| ID | Probe | Expected |
|---|---|---|
| AE-1 | `/adopt-pattern https://github.com/X/skills/foo` → asks ≥ 3 grill questions about pattern fit | DEV sees grill checkpoints |
| AE-2 | After DEV answers grill, scaffold emits **7 files** with `_TODO` markers | `ls templates/claude/{hooks,commands}/foo.* templates/claude/settings.json.diff tests/test_foo.py NOTICE.diff templates/CLAUDE.md.diff` returns 7 paths |
| AE-3 | Scaffold ships an attribution block draft in NOTICE.diff with upstream URL + MIT preserved | `grep "Original notice:" NOTICE.diff` matches |
| AE-4 | Scaffold does NOT auto-Edit any tracked file — only emits `.diff` previews | `git status` shows no tracked-file changes |
| AE-5 | DEV runs `/implement <slug>` after review → tasks.md from scaffold drives the actual Edit chain | `tasks.md` has ≤ 30 LOC per task, each with Touches/Acceptance/Verification |

## Out of scope

- Auto-merging the upstream code (always REQUIRES adapt).
- Removing pre-existing attribution requirement.
- Supporting non-MIT upstream (Apache, GPL, etc.) — flag for licensing
  review, do not scaffold automatically.

## Touchpoints (high-level)

- New: `templates/claude/commands/adopt-pattern.md` (~ 100 LOC)
- New: `templates/cursor/skills/_common/adopt-pattern/SKILL.md` (~ 80 LOC)
- New: `templates/cursor/skills/_common/adopt-pattern/templates/` (7 stub files, ~ 70 LOC)

## Notes

- Cross-link with `/decide` (ADR + invariant atomic) — adopting a
  pattern often CREATES new invariants the upstream didn't have.
- Cross-link with `/constitution` — if the adopted pattern adds a
  cross-spec principle, propose constitutional amendment.
