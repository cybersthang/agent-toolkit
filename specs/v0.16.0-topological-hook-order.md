# Spec: Topological hook order (replace hardcoded index)

## Problem

`tests/test_stop_chain_interactions.py` hardcodes hook ORDER + COUNT:

```python
assertEqual(names[0], "invariant_guard.py")
assertEqual(names[-1], "implement_snapshot_hook.py")
assertEqual(len(hooks), 10)    # was 9, was 8, was 7, ...
```

Every hook addition touches this test. Last time (v0.13.0
clarification_gate_enforcer) the test was left red. Last refactor
(v0.13.0+1 git_guardrails) needed manual settings.json reorder to
keep `invariant_guard.py` at `[0]` because test reads `PreToolUse[0]`.

This is **test ↔ JSON config ↔ code triangulation** — fragile, every
change costs context.

## Goal

Hooks declare their **ordering constraints** in a frontmatter docstring:

```python
"""... existing docstring ...

hook_order:
  required_before: [analyze_halt_gate, spec_first_guard]
  required_after:  []
  conflicts_with:  []
"""
```

`setup.py` reads each hook's `hook_order:` block at install time, runs
a topological sort, and emits `settings.json` chains in valid order.
Test asserts the ORDERING CONSTRAINTS hold, not the absolute index.

## Acceptance evals

| ID | Probe | Expected |
|---|---|---|
| AE-1 | `python setup.py validate-hook-order` — scans all hooks, runs topo sort | Returns valid ordering or `CYCLE: <hookA> ↔ <hookB>` |
| AE-2 | Add a 21st hook with `required_before: invariant_guard` → topo sort places it first | settings.json regenerated correctly |
| AE-3 | Add a hook with circular dep → installer FAILS with cycle report | Exit 2 + clear cycle path |
| AE-4 | `test_pretool_chain` rewrites: assert `invariant_guard.py` appears BEFORE `implement_snapshot_hook.py` in the same matcher group (not `== [0]` and `== [-1]`) | Hook count irrelevant to test |
| AE-5 | `test_stop_chain_length` removed entirely — replaced with constraint-based test | No test references `len(hooks) == N` |

## Migration plan

1. Add `hook_order:` block to each existing 21 hooks (one-line scan per file)
2. Implement topo sort helper in `lib/installer.py` (`_topo_sort_hooks`)
3. Update `templates/claude/settings.json` template — replace hardcoded
   array with `{{HOOK_CHAIN_PRETOOL_EDIT}}` / `{{HOOK_CHAIN_STOP}}` placeholders
4. `build_plan` renders the placeholders from topo result
5. Rewrite `test_stop_chain_interactions.py` to assert constraints
6. CHANGELOG: breaking change for anyone who hand-edits settings.json
   (now gets regenerated)

## Risks

- Hand-edit overlay friction: DEV who customized settings.json now has
  to override via a new `settings.override.json` mechanism (TBD).
- Increases setup.py size further (already 907 LOC over 800 budget).
  PRE-REQUISITE: split setup.py per loc-budget warning before doing this.

## Definition of done

- Adding a new hook requires zero edits to test_stop_chain_interactions.py
- Adding a hook with conflicting order requirement FAILS LOUDLY
- Hardcoded `names[0]` / `names[-1]` assertions are gone from tests
