# Spec: PostToolUse parallel-fire for independent hooks

## Problem

PostToolUse chain has 8 hooks (`probe_autostub` â†’ `tdd_runner` â†’
`verification_loop` â†’ `verify_nudge` â†’ `auto_test_runner` â†’
`auto_run_probes` â†’ `daemon_manager` â†’ `loc_delta_tracker`).

Worst-case sum of timeouts: **5 + 35 + 8 + 5 + 150 + 120 + 90 + 5 =
418 seconds = ~7 minutes** per Edit/Write. Even normal case sequential
(no timeout hit) is several seconds of pure overhead.

But these hooks are **largely independent**:

| Hook | Reads | Writes |
|---|---|---|
| `probe_autostub` | `acceptance-probes.json` | `.auto_probe_stub.json` |
| `tdd_runner` | `tdd.json`, source file | stdout nudge |
| `verification_loop` | `verification.json` | stdout nudge |
| `verify_nudge` | `specs/`, edit target | stdout nudge |
| `auto_test_runner` | `auto_test.json` | `.auto_test_state.json` |
| `auto_run_probes` | `acceptance-probes.json` | `.auto_run_probes_state.json` |
| `daemon_manager` | `test_env.json` | daemon PID file |
| `loc_delta_tracker` | git diff | `.loc_delta_log.json` |

Only `daemon_manager` â†” `auto_run_probes` have a real dependency
(probe wants daemon up). Rest can fire in parallel.

## Goal

Group PostToolUse hooks into **parallel-safe sets**, fire each set
concurrently. Total elapsed = max(set durations), not sum.

## Approach

Two options to evaluate:

### Option A: Claude Code native parallel-matcher
Claude Code's hook config may already support parallel-fire within one
matcher entry. Investigate first â€” if yes, just declare groups in
settings.json. ZERO toolkit-side code.

[verify: read Claude Code hook docs + open an issue with Anthropic if

### Option B: Toolkit-side fan-out
If Claude Code fires hooks sequentially regardless, wrap groups in
a launcher script that `subprocess.Popen` each member, waits, merges
stdout in declared order. New file: `templates/claude/hooks/_parallel_launcher.py`.

Cost: + 80 LOC launcher + complexity in stdout interleaving. Probably
not worth it if Option A works.

## Out of scope

- PreToolUse parallelization (block decisions need ordered evaluation).
