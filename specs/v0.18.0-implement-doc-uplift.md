# Spec: Auto-fire HTML + MD implement-doc sidecar after `/implement`

## Problem

Today's `/implement` auto-chain ends with `/verify` + báo cáo. Sidecar
`<slug>.implement-noted.md` is documented as "Phase 5.5 — Automatic
(preferred)" in [templates/cursor/skills/_common/implement-notes/SKILL.md:18](agent-toolkit/templates/cursor/skills/_common/implement-notes/SKILL.md),
but in practice it does NOT auto-fire. Only **5 of ~30 spec slugs**
have a `.implement-noted.md` companion (~17% adoption rate, sampled
2026-05-24 via `ls specs/*.implement-noted*`).

Plus: DEV review is faster in HTML than in raw Markdown. The composite
sprint `v0.13.x-lam-het-di-sprint.implement-noted.html` (this session)
proved the HTML rendering value — collapsible details, checkbox per
file, visual badges per severity.

## Goal

After `/implement` task chain finishes + `/verify` emits report, agent
**automatically** generates BOTH:

- `<slug>.implement-noted.md` — machine-parseable, validator-friendly
- `<slug>.implement-noted.html` — DEV-readable in browser, self-contained
  (embedded CSS, no external dep)

DEV reads the HTML, ticks checkboxes, decides merge. Zero manual
`/implement-notes` invocation.

## User stories

- **US1**: As DEV, after `/implement <slug>` finishes, I get `<slug>.implement-noted.html` automatically. I don't need to remember to run `/implement-notes`.
- **US2**: As DEV reviewing the doc, I see scope deviations (SD), in-transcript trade-offs (T), follow-ups (F), confidence summary — same 4-section schema as existing MD template — rendered with visual hierarchy (badges per severity, collapsible file list, clickable anchors).
- **US3**: As DEV, I can disable auto-fire per project via `.agent-toolkit/implement_notes.json` `"auto_emit": false`. Default `true` for new installs.
- **US4**: As AGENT, `/implement-notes <slug>` slash command accepts `--format md|html|both` flag (default `both`), overrides project config.
- **US5**: As DEV, if I want only HTML (skip MD), I configure `"output_format": "html"` in project config.

## Out of scope

- Other formats (PDF, JSON sidecar) — defer to future spec.
- Auto-commit the sidecar — DEV reviews + commits manually.
- Rendering pipeline using external tool (pandoc, etc.) — must be Python-stdlib only to match toolkit's zero-dep contract.
- Diff annotations (current Phase 5.3 of `implement_orchestrator.py` continues to work; sidecar lists SD-N, doesn't replace annotations).

## Risks

- **R1**: HTML template diverges from MD template over time. Mitigation: keep schema sections identical (§1-4), only rendering differs. Both files generated from same data extraction pass.
- **R2**: Existing 5 spec sidecars (`.md` only) don't get HTML retroactively. Mitigation: provide `/implement-notes <slug> --format html` for manual retro-emit. Document in v0.18 release notes.
- **R3**: HTML file size — composite sprint HTML was 483 LOC. Per-spec HTML target ≤ 300 LOC (smaller scope, fewer items). Mitigation: HTML template lean, no heavy CSS.
