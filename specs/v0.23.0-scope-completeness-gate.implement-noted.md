## §2 In-transcript trade-offs (decisions made this session)

- **Block via exit code 2, not `{"decision":"block"}` envelope.** Chose to
  mirror `gap_completeness_gate` exactly (exit 2) rather than the
  `emit_stop_block` JSON envelope used by evidence_audit/verify_lint. Reason:
  sibling-parity + the chain-interaction test's `blocking_stop_hooks`
  allow-list doesn't include the gap/scope gates (they block by exit code).
  Consequence: spec eval us2's drafted assertion ("source contains
  `\"decision\": \"block\"`") was replaced with `returncode == 2`.
- **`DONE_FULL_CLAIM_RE` is broader than `DONE_CLAIM_GAP_RE`.** Added
  standalone completion words (`done`/`verified`/`finished` followed by
  punctuation) because the manifest enumerates the FULL upfront scope, so
  any completion phrasing should gate. Trade-off: slightly higher
  false-positive surface than the gap gate — mitigated by the activation
  boundary (only fires when a manifest exists).
- **Gate does NOT skip on autonomy active** (opposite of gap gate). The R9
  bug is partial-done during a multi-item run, so the scope gate must be
  live during autonomy. Intermediate per-task "T1 passed" claims don't match
  `DONE_FULL_CLAIM_RE`, so mid-chain work isn't blocked — only a full/done
  claim with pending items.

## §3 Open follow-ups

- **F1 (low)**: `implement_orchestrator.py` could PRE-create the manifest at
  autonomy start (eager) so the manifest is visible from turn 1 rather than
  first-Stop. Current lazy-derive is functionally equivalent for the gate
  but the manifest file appears one turn later. Candidate for a v0.24 polish.
- **F2 (low)**: `_task_status_to_item` maps `skipped` → `deferred`. The
  tasks-breakdown skill says skipped tasks "count as GAP in /verify". If a
  future policy wants skipped to remain `pending` (force re-decision),
  revisit this mapping.
- **F3 (low)**: ad-hoc manifest reads only the LATEST TodoWrite call. If an
  agent shrinks its todo list mid-session, the manifest reflects the latest
  snapshot only (no historical union). Acceptable for the R9 case; note for
  future if cross-turn scope accumulation is desired.
