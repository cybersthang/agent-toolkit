# Spec — scope-completeness-gate (R9)

## 1. Problem

Session 2026-05-27: DEV gõ "làm full" (fix ALL reviewer findings — multi-item
scope). Agent làm 7/14, để 4 partial + 3 deferred, claim substantial progress.
DEV phải hỏi tay "đã làm đầy đủ chưa" → agent MỚI audit (reactive). Expected:
agent tự audit completeness NGAY khi claim done.

**Root cause (verified)**: `gap_completeness_gate.py` (templates/claude/hooks/
gap_completeness_gate.py:5,135,154) đo completeness theo gaps mà AGENT tự
surface qua `G<N>` pattern vào `.open_gaps.json`. Multi-item request KHÔNG
được enumerate upfront thành mechanical checklist. Partial completion → vô
hình với gate vì các item chưa làm không bao giờ register thành gap. Self-audit
optional/reactive, không gate-enforced.

**Phân biệt với gap_completeness_gate**:
- gap gate: tracks gaps agent SURFACE giữa chừng (reactive, chống drip-feed)
- scope gate (THIẾU): tracks REQUEST scope khai báo UPFRONT (proactive, chống partial-done)

## 2. Solution

Mechanical scope-completeness gate đóng đúng lỗ hổng:

1. **Scope manifest** derive từ STRUCTURED artifact (KHÔNG parse DEV prompt):
   - Priority 1: `tasks.md` (Spec Kit) — mỗi T<N> = 1 manifest item
   - Priority 2: `acceptance_evals` (spec frontmatter) — mỗi eval = 1 item
   - Priority 3: ad-hoc — agent tự emit manifest khi ≥N self-generated items
     (TodoWrite list / sub-agent batch)
   - Persist: `.agent-toolkit/.scope_manifest.json`

2. **Stop hook `scope_completeness_gate.py`**: trước done/full-claim, verify
   mọi manifest item status ∈ {done, deferred:<reason>, cant:<reason>}.
   Còn `pending`/`partial` không defer → BLOCK + list incomplete.

3. **Resolution markers** (như gap gate): `scope-done: S<N>`,
   `scope-defer: S<N> <reason>`, `scope-cant: S<N> <reason>` trong response.

4. **Activation = autonomy active + manifest exists**, KHÔNG prompt keyword.
   Request nhỏ (no manifest) → gate silent → zero friction.

- **Story 1**: Manifest derive từ tasks.md (Spec Kit autonomy), KHÔNG regex DEV prompt.
- **Story 2**: Stop hook BLOCK done-claim khi manifest còn pending item.
- **Story 3**: Resolution markers (scope-done/defer/cant) resolve items.
- **Story 4**: Activation boundary — gate silent khi không manifest (zero friction request nhỏ).
- **Story 5**: Ad-hoc multi-item — agent tự emit manifest khi ≥N self-generated items.

## 5. Implementation Decisions

Chốt ở /clarify 2026-05-27 (Q1/Q2 DEV chốt · Q3/Q4/Q5 self-resolve):

- **D1 (Q1 — ad-hoc source)**: Khi không có tasks.md, manifest auto-emit
  từ **TodoWrite list ≥3 items** (mỗi todo = 1 item S<N>). Sub-agent batch
  KHÔNG auto-trigger. <3 items → không emit. Source priority cuối:
  `tasks.md → acceptance_evals → TodoWrite≥3`.
  *Why*: TodoWrite là tín hiệu mechanical đúng nhất, cover đúng case session
  2026-05-27 "fix all" fail mà không over-engineer; ngưỡng 3 mirror gap gate.

- **D2 (Q2 — block vs warn)**: Default mode = **block** (mạnh như
  gap_completeness_gate). Done/full-claim + manifest còn pending → BLOCK +
  list incomplete. Promote/demote qua `.agent-toolkit/enforce_mode.json`
  `per_hook.scope_completeness_gate`.
  *Why*: đóng đúng root cause R9 (partial-done pass silently); khớp tiến
  thần "làm full" + sibling gap gate cũng block. Bypass token cho exception.

- **D3 (Q3 — lifecycle, self-resolved)**: Manifest lifecycle **mirror
  autonomy**: create khi autonomy start (`.autonomy_active.json` ghi) ·
  clear khi autonomy off HOẶC /verify pass · TTL = autonomy `expires_at`.
  *Why*: tránh manifest mồ côi; autonomy đã là boundary của multi-step work.

- **D4 (Q4 — tasks.md sync, self-resolved)**: Gate đọc **trực tiếp tasks.md
  task status** (`passed | failed | skipped`, đã có ở implement.md:105) làm
  manifest item status — KHÔNG cần separate sync layer. Single source of
  truth, zero drift.

- **D5 (Q5 — bypass, self-resolved)**: Whole-gate bypass token
  `bypass-scope-gate: <reason ≥8 chars>` trong prompt, single-shot via
  `.skip_scope_gate_next.json` (TTL 600s) — mirror 4 sibling bypass tokens
  trong `_patterns.py` (bypass-invariant / gap-gate / git-guard / debug-sentry).
