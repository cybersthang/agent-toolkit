## Deviations from spec (carried to implement-notes)

1. **us2 envelope form**: spec eval text said source contains
   `"decision": "block"`. Implemented as **exit code 2** (mirrors sibling
   `gap_completeness_gate`, which Claude Code treats as a Stop block).
   Test asserts `returncode == 2`, not the JSON envelope.
2. **Manifest derive location**: consolidated into the gate hook (lazy
   derive on first Stop while autonomy active) rather than editing
   `implement_orchestrator.py`. Spec §3 allowed "implement_orchestrator
   HOẶC /implement skill"; lazy-in-gate is lower-risk + fully testable.
3. **TodoWrite capture**: reads the session transcript (`transcript_path`)
   for the latest TodoWrite call — no new PostToolUse hook, no settings
   churn (T3 originally implied a snapshot mechanism).
4. **Eval probe node names**: actual test method names differ slightly from
   the spec's drafted node strings (e.g. `test_silent_when_no_manifest`
   → `test_silent_when_no_manifest_no_autonomy`). Same behavior, mapped by
   test class above.
