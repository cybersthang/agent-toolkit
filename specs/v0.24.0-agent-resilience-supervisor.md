# Spec — agent-resilience-supervisor (re-scoped: stall-watcher + notify)

## 1. Problem

Khi agent (hoặc sub-agent song song) chạy autonomous multi-step work, 2 sự
cố làm gãy phiên + mất tiến độ:
1. **`API Error: 529 Overloaded`** — lỗi tầng API lúc model generate.
2. **AGENT treo** — phiên đơ giữa chừng (tool deadlock, chờ stdin, API stall).

DEV mong: tự phục hồi, hoặc **ít nhất phát hiện được và báo DEV** để DEV vào
chat gõ "tiếp".

**Ràng buộc kiến trúc đã verify (2026-05-27, qua claude-code-guide + quan sát phiên):**
- Hook chỉ fire ở 5 event; khi 529/treo xảy ra **không event nào fire** →
  hook KHÔNG tự detect/relaunch được. Tiến trình treo không tự watchdog mình.
- **DEV dùng Claude Code qua chat-VSCode-extension** → phiên do VSCode quản,
  **KHÔNG có CLI process để wrap** → external supervisor full-auto KHÔNG áp
  dụng được cho extension (chỉ áp dụng path CLI/headless).
- Claude Code **đã tự retry 529 nội bộ 10×** (`CLAUDE_CODE_MAX_RETRIES`,
  exponential backoff) rồi mới exit. Không có fallback-model setting.
- Transcript ghi tăng dần ở `~/.claude/projects/<proj>/<session-id>.jsonl`
  (per message/tool) → resumable; extension có Session History UI.
- Khi 529 cạn / treo trong extension: **không có nút Retry/Stop** (issue
  #39381) → hiện phải kill + mất việc, hoặc resume thủ công.

**Gap**: chưa có (a) lớp phát hiện treo + báo DEV chạy được cho cả extension,
(b) core đọc trạng thái "đang dở ở đâu" để resume idempotent, (c) auto-relaunch
cho path CLI.

## 2. Solution

Pivot khỏi "external full-auto supervisor" (bất khả thi cho extension) sang
**watcher read-only + notify**, cộng auto-relaunch chỉ cho CLI. Một core
dùng chung, tận dụng R9 manifest làm checkpoint.

```
┌── resume_state.py (CORE) ── đọc .scope_manifest.json + tasks.md + autonomy
│      → sinh "resume brief" (đang dở T<i>, pending S<n>)   [dùng chung]
│
├── stall_watcher (tools/agent_supervisor.py, NGOÀI, READ-ONLY) ─────────┐
│     theo dõi transcript active (mtime) + autonomy + process-alive       │
│     stale > ngưỡng (hoặc process chết) → NOTIFY DEV                      │ ← cả extension + CLI
│     KHÔNG wrap, KHÔNG kill, KHÔNG sửa session  (an toàn, false-alarm rẻ) │
│        └─ chế độ --relaunch (CLI-only): claude -c -p "<resume brief>"    │
│           tối đa 10 lần + backoff; cạn cap → notify DEV                  │
│
├── notify (pluggable): desktop toast + log(always) + email(SMTP)         │
│      + webhook(Slack/Discord); creds qua .codex/mcp.local.env           │
│
└── doc/config: CLAUDE_CODE_MAX_RETRIES (giảm 529 từ gốc, cả extension)
```

- **Extension**: watcher phát hiện treo → báo DEV → DEV vào chat gõ "tiếp" →
  agent đọc manifest (resume_state) → tiếp idempotent. (semi-auto, đúng trần)
- **CLI**: watcher `--relaunch` tự `claude -c -p` cap 10 → cạn → notify. (full-auto)

## 4. User Stories

- **Story 1**: `resume_state` core — đọc R9 manifest+tasks+autonomy → resume brief.
- **Story 2**: `stall_watcher` read-only — detect treo (transcript stale + autonomy active / process chết) → trigger notify. Chạy cả extension+CLI, không wrap/kill.
- **Story 3**: notify pluggable — toast + log(always) + email(SMTP) + webhook(Slack/Discord); creds qua mcp.local.env.
- **Story 4**: CLI auto-relaunch — `claude -c -p "<brief>"` cap 10 + backoff; cạn → notify. CLI-only.
- **Story 5**: resume idempotent — DEV "tiếp" / relaunch prompt → agent đọc manifest → tiếp pending, không redo passed.
- **Story 6**: cross-platform Win+Ubuntu (watcher+notify+relaunch) + doc CLAUDE_CODE_MAX_RETRIES.

## 5. Implementation Decisions (chốt ở /clarify 2026-05-27)

- **D1 (architecture pivot)**: watcher READ-ONLY + notify, KHÔNG wrap/kill session. Lý do: extension không wrap được; read-only chạy được cả 2 path + false-alarm rẻ (notify ≠ kill).
- **D2 (recovery level)**: extension = semi-auto (DEV gõ "tiếp"); CLI = full-auto relaunch cap 10 + backoff → cạn → notify. Trần của extension là semi-auto (giới hạn nền tảng, không vá được).
- **D3 (checkpoint)**: tái dùng R9 `.scope_manifest.json` + tasks.md status làm trạng thái idempotent — không thêm checkpoint mới.
- **D4 (notify channels)**: pluggable, mặc định bật toast + log; email(SMTP) + webhook(Slack/Discord) opt-in; creds qua `.codex/mcp.local.env` (credentials policy — không commit).
- **D5 (529)**: KHÔNG đấu với retry nội bộ; doc/config `CLAUDE_CODE_MAX_RETRIES` cao hơn; watcher chỉ hành động khi claude đã exit (CLI) / transcript stale (extension).
- **D6 (process-alive)**: `psutil` check process còn sống (watcher-only dep); thiếu psutil → degrade về transcript-mtime-only.
- **D7 (Q1 stall threshold, self-resolved)**: default fixed generous **180s** (config `stall_seconds`), optional adaptive = p99 của `.hook_fire_log.json` × 3 (sàn 120s). Vì notify-only nên để rộng, false-alarm rẻ.
- **D8 (Q2 active-transcript, self-resolved)**: watcher chọn file `.jsonl` mtime mới nhất trong `~/.claude/projects/<encoded-proj>/`. Đủ cho phiên đơn; multi-session để Q4 quyết.
- **D9 (Q3 hook-on-resume, self-resolved)**: verify thực nghiệm ở /implement (task riêng). Fallback an toàn nếu SessionStart không fire on resume: `.scope_manifest.json` vẫn trên đĩa → agent đọc khi DEV mở lại; resume-brief hook chỉ là tự-động-hoá, không load-bearing.
- **D10 (Q5 notify dedupe, self-resolved)**: báo lần đầu khi phát hiện stale, sau đó **lặp lại mỗi cooldown** (config `notify_cooldown`, default 300s) tới khi transcript tăng lại / DEV xử lý. Dedupe theo session-id để không spam mail/webhook.
- **D11 (Q4 sub-agent scope, DEV chốt)**: v0.24 watcher CHỈ theo dõi **phiên Claude chính** (1 transcript = file `.jsonl` mtime mới nhất). Multi-agent/sub-agent monitoring để spec `parallel-subagent-guard` (spec 2) lo — tách concern, ship sớm.
