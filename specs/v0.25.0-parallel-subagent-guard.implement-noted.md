# Implement Notes — parallel-subagent-guard (v0.25.0)

## §1 Scope deviations

1. **Hook placement in PreToolUse chain**: inserted right after
   `invariant_guard` (semantic sibling — both DENY-tier guards). Spec §3
   said only "wire into PreToolUse" without exact position. Pre-existing
   chain-order test asserts only first/last, so the insertion is safe.
2. **CLI helper in `tools/`**: `parallel_wave.py` placed at the toolkit's
   `tools/` (matches v0.24 `agent_supervisor.py` precedent), NOT under
   the installed `.claude/`. Rationale: it's a DEV-invoked helper, not a
   hook; consumers run it via `python tools/parallel_wave.py` from the
   toolkit repo OR the toolkit installer can ship it to consumer in a
   future v0.26 polish.
3. **PreToolUse block envelope shape**: used new
   `{"hookSpecificOutput": {"hookEventName":"PreToolUse",
   "permissionDecision":"deny", ...}}` instead of the older exit-code-2
   pattern Stop-chain blockers (scope/gap/etc.) use. Matches PreToolUse
   contract per hooks.md and tests confirm `payload["hookSpecificOutput"]
   ["permissionDecision"]=="deny"`.

## §2 In-transcript trade-offs

- **Identity model = `agent_id` from envelope.** Verified via
  claude-code-guide that the field IS in PreToolUse envelopes for
  sub-agents ("Only in subagents" per hooks.md). Main agent → field
  absent. The guard treats `None != some_id` as cross-zone, which makes
  the D8 cross rule (main↔sub also blocks) work without extra logic.
- **Smart-match (D4) for owned patterns**: chose single field that
  resolves to glob (if `*`/`?`/`[`), dir-prefix (trailing `/`), else
  exact path. Simpler schema than 3 separate fields.
- **Relaunch cap / detect treo OUT-OF-SCOPE**: that's v0.24's
  `agent_supervisor`. This spec is pure conflict-avoidance, not
  resilience.
- **WSL Linux verified before declaring done** (lesson from v0.24
  dispatch monkeypatch cache bug that passed Windows but failed Linux
  CI). Tests use Path.resolve + fnmatch — both cross-platform.

## §3 Open follow-ups

- **F1 (MEDIUM — load-bearing)**: empirically verify PreToolUse hooks
  ACTUALLY fire for sub-agent Edit calls (docs gap, spec D1). If they
  don't, the guard only catches main-agent cross-zone but not sub↔sub.
  Test on a real `Agent` tool fan-out before relying on it for
  production parallel safety.
- **F2 (LOW)**: empirically verify the exact field name (`agent_id`
  vs `subagent_id` vs other) and stability across Claude Code releases.
  If shape changes, the guard's identity check breaks silently (would
  treat as "no agent_id" → all sub-agents look like main agent).
- **F3 (LOW)**: installer doesn't currently ship `tools/parallel_wave.py`
  to consumer `.claude/`. DEV using consumer project must run from
  toolkit dir; v0.26 polish: copy `tools/` into consumer or expose via
  a slash command.
- **F4 (LOW)**: skill SKILL.md lists `wave-a-1` etc. as example IDs; if
  agent picks generic IDs like `agent-1` always, two waves running back
  to back could confuse — recommend wave-prefixed IDs in skill body
  (already mentioned but could be stricter).
