# Spec — parallel-subagent-guard (v0.25.0)

## 1. Problem

DEV yêu cầu (2026-05-27): *"thêm tính năng cho phép Agent tạo ra các
Sub-agent làm việc song song với những task có thể làm việc song song mà
KHÔNG conflict với nhau"*.

Sự thật kỹ thuật cốt lõi (đã verify qua phiên v0.24 + qua chính cách
session này dùng Agent tool ở Wave A):
- Toolkit **KHÔNG tự "tạo" sub-agent** — `Agent tool` là cơ chế của
  Claude Code harness, chỉ chính model gọi. Toolkit không ép model spawn.
- Cái toolkit LÀM ĐƯỢC (đúng triết lý mechanical enforcement): **chặn
  conflict cơ học** khi nhiều sub-agent song song chạm cùng file.
- Pattern an toàn đã chứng minh khả thi: **file-disjoint Wave** (mỗi
  sub-agent sở hữu tập file rời nhau) — chính cách tôi chạy Wave A v0.21
  parallel (file disjoint, 0 conflict).

Toolkit hiện có:
- `invariant_guard` PreToolUse (DENY edit phá invariant) — pattern check
  per-Edit nhưng KHÔNG quan tâm tới agent identity / concurrent ownership.
- `scope_completeness_gate` (R9) — manifest pattern (`.scope_manifest.json`)
  cho tracked scope — precedent cho manifest-driven gate.
- Wave/parallel pattern hoàn toàn ở mức skill/agent-discretion: không có
  cơ chế cơ học chặn 2 sub-agent đụng cùng file.

Gap: **không có lớp guard "ai sở hữu file nào trong wave này"** + không có
manifest/skill hướng dẫn pattern an toàn. Conflict cross-zone hiện dựa
hoàn toàn vào agent tự kỷ luật.

## 2. Solution

Hai lớp, tận dụng pattern manifest đã có ở R9/v0.24:

**Lớp 1 — Skill `parallel-batching`** (guide, không enforce): hướng dẫn
agent **trước khi gọi Agent tool**:
1. Phân chia task thành các zone file-disjoint (mỗi sub-agent 1 zone).
2. Gọi 1 helper (CLI tool / hook command) emit manifest
   `.agent-toolkit/.parallel_wave.json` với cấu trúc agent_id → owned_paths.
3. Sau khi sub-agent batch xong, declare wave-done → clear manifest.

**Lớp 2 — `parallel_conflict_guard.py` PreToolUse hook**: với mỗi Edit/
Write/MultiEdit, nếu manifest tồn tại:
1. Resolve owner zone của file target.
2. Resolve identity của agent đang edit (qua envelope — cách EXACT cần
   verify ở /clarify, xem Open Questions Q1).
3. Cross-zone (file thuộc owner khác) → BLOCK với reason rõ. Same-owner
   → allow. Outside any zone → silent allow (zero-friction).

Manifest schema (đề xuất, finalize ở /clarify):
```json
{
  "version": 1,
  "wave": "<name>",
  "created_ts": ...,
  "ttl_seconds": 3600,
  "zones": [
    {"agent_id": "<id-or-session>", "owned": ["templates/claude/hooks/foo.py", "tests/test_foo.py"]},
    {"agent_id": "...", "owned": ["templates/agent_toolkit/bar.json"]}
  ],
  "wave_done": false
}
```

Lifecycle (mirror R9 manifest + autonomy):
- Create: khi skill `parallel-batching` emit manifest qua helper.
- Clear: `wave_done=true` HOẶC autonomy off HOẶC TTL hết (default 1h).

## 5. Implementation Decisions

Chốt ở /clarify 2026-05-28 (Q1 verified qua claude-code-guide; Q2-Q7 self-resolved):

- **D1 (Q1 — agent identity, verified)**: PreToolUse envelope CÓ `agent_id` +
  `agent_type` cho sub-agent (chính thức ở docs `code.claude.com/docs/en/hooks.md`
  "Common Input Fields", marked "Only in subagents"). Guard dùng 2 field này.
  Main agent → 2 field vắng / khác. Empirical-verify: docs KHÔNG xác nhận rõ
  PreToolUse CÓ fire cho sub-agent Edit không (docs gap) — verify ở /implement
  bằng test integration thật; plan B nếu không fire = dùng `SubagentStart`
  event snapshot ownership lúc spawn.

- **D2 (Q2 — manifest emission)**: Primary path = **CLI helper**
  `tools/parallel_wave.py emit/declare-done/clear` (main agent gọi trước khi
  Agent tool spawn). Optional reinforcement: `SubagentStart` hook auto-emit
  zone entry nếu agent prompt khai báo ownership. CLI đơn giản, mechanical,
  agent dễ nhớ — bám pattern của `tools/agent_supervisor.py` v0.24.

- **D3 (Q3 — block vs warn)**: Default **`block`** (mirror `invariant_guard` +
  `scope_completeness_gate` + `gap_completeness_gate` — đều mechanical-enforcement
  tier). Configurable qua `.agent-toolkit/enforce_mode.json` `per_hook.parallel_conflict_guard`.

- **D4 (Q4 — zone matching mode)**: Support **cả 3** qua 1 field thống nhất với
  smart-resolve:
  - Có `*` → glob (fnmatch).
  - Kết thúc `/` → dir-prefix.
  - Khác → exact path.
  Cho phép DEV linh hoạt mà không tăng schema complexity.

- **D5 (Q5 — lifecycle)**:
  - **Create**: `parallel_wave.py emit ...` (main agent) HOẶC `SubagentStart`
    hook (auto, optional).
  - **Clear**: 4 trigger — (a) `parallel_wave.py clear` declare wave-done,
    (b) `SubagentStop` cho mọi zone → file xoá, (c) TTL hết (default 1h),
    (d) autonomy off.
  - **Bypass single-shot**: `bypass-parallel-guard: <reason ≥ 8 chars>` trong
    prompt → intent_router ghi `.skip_parallel_guard_next.json` (TTL 600s,
    mirror sibling tokens trong `_patterns.py`).

- **D6 (Q6 — scope)**: Phase 1 = **manifest-only** (zero false-positive nếu
  không có manifest). Auto-detect concurrent edit không manifest = scope v0.26+.

- **D7 (Q7 — codify Wave pattern)**: Skill `parallel-batching` SKILL.md ship
  với **template cụ thể**: (1) plan zones file-disjoint, (2) CLI emit
  manifest, (3) gọi N Agent tool song song, (4) chờ xong, (5) CLI clear.
  Dogfood từ Wave A v0.21 (đã chạy thật, 0 conflict).

- **D8 (cross-edit rule)**: Cross-zone block dùng cho **cả main↔sub lẫn sub↔sub**.
  Rule đơn giản: Edit file F mà F ∈ zone Z và `envelope.agent_id != Z.agent_id`
  → BLOCK. Main agent edit zone của sub-agent KHI sub-agent đang chạy = race
  condition, phải block. Outside-zone edit luôn allow.

## 6. Testing

Mỗi Story → 1 acceptance_eval. Guard test = mock PreToolUse envelope
(tool_input.file_path) + seed manifest → assert decision (block/allow/silent).
Skill smoke test = lint markdown. Cross-platform: CI matrix Win+Ubuntu xanh
cho test file. Helper CLI test = subprocess `tools/parallel_wave.py emit/
declare-done/clear` → assert manifest state.

## 7. Out-of-scope

- KHÔNG ép model spawn sub-agent — đó là quyết định của model, không
  enforce được. Skill chỉ guide.
- KHÔNG quản multi-process locking (race condition giữa 2 sub-agent edit
  cùng lúc cùng file SAME-ZONE): nằm ngoài scope; manifest cross-zone là
  đủ cho mục tiêu DEV.
- KHÔNG kết hợp với `agent_supervisor.py` (v0.24) ở phase 1; supervisor +
  guard là 2 concern riêng (resilience vs conflict).
- KHÔNG dynamic re-balance zone giữa chừng — wave-done → re-emit cho wave
  mới.
