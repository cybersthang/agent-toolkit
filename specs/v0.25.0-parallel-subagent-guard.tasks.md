## T3 — parallel_conflict_guard.py PreToolUse hook

- **Touches:** `templates/claude/hooks/parallel_conflict_guard.py` (new ~180 LOC)
- **Depends on:** T1, T2
- **Acceptance:** Covers `us3-conflict-guard-block` + `us4-same-owner-allow` +
  `us5-outside-zone-silent` + `us6-manifest-lifecycle`. Logic:
  1. Read `.parallel_wave.json`; missing/TTL-expired/wave_done → exit 0 silent
     (us5+us6).
  2. Resolve target file from `envelope.tool_input.file_path` (Edit/Write/MultiEdit).
  3. Find owning zone via D4 smart-match (glob/dir-prefix/exact).
  4. If file ∉ any zone → exit 0 silent (us5).
  5. Identity: `envelope.agent_id` (sub-agent) or None (main agent).
  6. D8 rule: `zone.agent_id != envelope.agent_id` → BLOCK with reason
     listing owner; same → allow (us3, us4).
  7. Bypass: `.skip_parallel_guard_next.json` fresh → consume + allow once (us6).
  8. Kill-switch `AGENT_TOOLKIT_DISABLE=1` honored; `stop_hook_active`
     recursion guard N/A (PreToolUse, not Stop). `get_enforce_mode` default
     `block` (D3); `warn`/`off` honored.
- **Verification:** `pytest tests/test_parallel_conflict_guard.py::TestGuard::test_cross_zone_block` + `::test_same_owner_allow` + `::test_outside_zone_silent` + `::TestLifecycle::test_clear_and_ttl`.
- **Risk:** medium — primary blocking hook; path normalize + glob cross-platform.
