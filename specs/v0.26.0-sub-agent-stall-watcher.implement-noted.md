## §2 In-transcript trade-offs

- **Phase 1 mapping skipped**: we don't bridge DEV-chosen `agent_id`
  (e.g. `"agent-a"`) ↔ Claude-generated `<session-id>.jsonl` filenames.
  Notify includes the transcript path so DEV opens it and correlates.
  Mapping enrichment deferred to v0.27 (F1 below). Decision driven by
  claude-code-guide finding: the bridge would require parsing the main
  transcript's Agent tool result for `agentId` + matching prompt strings —
  fragile.
- **Aggregate notify per tick (D2)** over per-sub-agent: less spam in
  the worst case (3 sub-agents all stall → 1 toast, not 3). Cooldown is
  still per-transcript (D3) so different sub-agents notify independently
  across ticks.
- **Notify-only for sub-agents (D8)**: confirmed Agent tool is model-
  invoked — toolkit has no API to re-spawn. v0.24 main-session
  `--relaunch` stays untouched.

## §3 Open follow-ups

- **F1 (MEDIUM — load-bearing-ish)**: enrich notify with `agent_id` DEV
  chose (e.g. "agent-a") instead of just the transcript filename.
  Requires parsing main transcript Agent tool results (`agentId` field
  per Claude Code docs) + string-match prompt to manifest `zones[]
  .agent_id`. Phase 1 skipped — fragile prompt-text matching.
- **F2 (MEDIUM — uncertain docs)**: empirically verify the EXACT path
  encoding + filename pattern Claude Code uses for sub-agent transcripts.
  Docs (sessions.md) say "separate files in same `<project>` dir" but
  don't confirm path layout for concurrent sub-agents. Run a 3-way
  parallel `Agent` fan-out on the target machine + `ls
  ~/.claude/projects/<encoded>/` to confirm.
- **F3 (LOW)**: concurrent-write contention not documented. If 3
  sub-agents write `.jsonl` concurrently and our watcher tail-reads, we
  may see partial lines occasionally. Phase 1 only checks `mtime`, never
  reads file CONTENT — so contention is moot for stall detection. F1
  (content parsing for agent_id) would need to handle it.
- **F4 (LOW)**: `subagent_stall_seconds` defaults to main `stall_seconds`
  (180s). Sub-agents typically do shorter tasks — DEV may want to drop
  to 60-90s. Mention in docs as tunable.
