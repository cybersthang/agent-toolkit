# v0.29.0 ‚Äî auto-parallel task waves for /implement

## 1. Problem / DEV ask

`/implement` shipped **sequential-only** (`implement.md`: "Execute tasks ‚Äî
sequential, NOT parallel"). The Q3 audit confirmed the gap: tasks.md already
carries `Touches` (files) and `Depends on` (edges), and
`parallel_conflict_guard` already enforces file-disjoint zones ‚Äî but nothing
turned that metadata into actual parallel execution. Independent tasks that
could run 3-4√ó faster ran one-by-one.

## 2. Scope

In:
- A deterministic **wave planner** (`tools/wave_planner.py`) that turns a
  tasks.md into ordered waves of provably file-disjoint, dependency-ready
  tasks, and can `emit` a `.parallel_wave.json` per wave.
- `/implement` wired to dispatch ‚â•2-task waves as concurrent sub-agents,
  falling back to the existing sequential loop otherwise.
- `/tasks` surfaces the wave preview at the DEV review gate.

Out (deliberately):
- No ML/heuristic guessing of independence ‚Äî only DECLARED `Touches` +
  `Depends on`. Anything not provably disjoint runs sequentially.
- No change to `parallel_conflict_guard` / `parallel_wave.py` (reused as-is).
- No auto-commit by sub-agents (git_guardrails unchanged).

## 3. Design

- `parse_tasks(md)` ‚Äî `## T<N>` headers + `**Touches:**` (multi-line, strips
  backticks + `(extend)`/`(new)` annotations) + `**Depends on:**` (`T<N>`).
- `plan_waves(tasks)` ‚Äî topological-level grouping; within each level, greedily
  pack tasks whose `Touches` are disjoint into one wave, defer conflicts to a
  later wave. **Conservative `_conflict`**: empty Touches, any glob (`* ? [`),
  or any shared path ‚áí NOT parallelized. Dependency cycle ‚áí full sequential
  fallback.
- `emit_wave(...)` ‚Äî reuses `parallel_wave.emit`; one zone per task
  (`agent_id = T<N>`, `owned = Touches`) so `parallel_conflict_guard` denies
  any cross-zone Edit during the wave.
- `/implement` step 9: plan ‚Üí for each wave, 1-task = inline, ‚â•2-task = emit
  manifest + spawn one sub-agent per task in a single message + wait +
  declare-done. Per-task FAIL handling (retry/skip/abort) unchanged.
