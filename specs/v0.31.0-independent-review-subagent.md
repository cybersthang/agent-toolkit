# v0.31.0 — Independent-review sub-agent (review độc lập ở done-boundary)

## 1. Problem

`/review` hiện chạy **trong cùng context** của agent đã viết code
([commands/review.md:1-13](../templates/claude/commands/review.md#L1) — allowed-tools
Read/Edit/Glob/Grep/Bash, MANUAL/opt-in). Reviewer cùng-context **nhiễm ngữ
cảnh (confirmation bias)**: nó mang sẵn lý lẽ/giả định đã sinh ra code nên
"hợp lý hoá" bug thay vì bắt. DEV quan sát thực tế: **mở session mới rồi
review thì ra rất nhiều BLOCKER** mà review cùng-phiên bỏ sót.

Eval nội bộ cũng chỉ ra: `/review` + `/implement-notes` là 2 bước
chất-lượng-cao-nhất nhưng **ít bị ép nhất** (review manual, notes WARN). Không
có hook nào enforce "đã review độc lập trước khi done".

## 2. Solution

Thêm một **reviewer sub-agent context-sạch** chạy ở **done-boundary** (sau
`/verify` pass, trước khi AGENT claim done), trả BLOCKER về luồng hiện tại để
fix, lặp tới khi hội tụ. Cơ chế bám đúng trust-model toolkit: **không tin
prose, verify bằng transcript + sha**.

3 lớp:
- **Lớp 1 — packet do CODE lắp:** CLI `tools/independent_review.py emit-context`
  gom `git diff` feature-scope + spec + acceptance_evals + invariants liên quan
  thành 1 packet + `packet-sha` (bám pattern
  [tools/parallel_wave.py:63,120](../tools/parallel_wave.py#L63)). Input của
  reviewer là **deterministic, do máy sinh** → độc lập by-construction (đóng C2).
- **Lớp 2 — agent spawn reviewer (Task tool) với CHỈ packet:** skill
  `independent-review` quy định prompt template; reviewer trả findings
  (BLOCKER/MEDIUM/LOW + Proof + Doubt-pass) → ghi `.independent_review.json`
  (key theo diff-sha).
- **Lớp 3 — gate verify bằng chứng un-fakeable:** `independent_review_gate.py`
  (Stop) chặn done-claim trừ khi: (a) có `tool_use` Task thật trong cycle, (b)
  sub-agent transcript tồn tại (reuse
  [agent_supervisor.discover_sub_agent_transcripts](../tools/agent_supervisor.py#L436))
  và **tiêu thụ packet-sha**, (c) artifact khớp **diff-sha hiện tại** + findings
  trỏ file:line có thật trong diff. → agent không thể tự gõ artifact mà bỏ qua
  spawn (đóng C1). Mô hình giống
  [pass_contract.py:168-170](../templates/claude/hooks/_audit/pass_contract.py#L168)
  (verify sha256 của tool_result, không tin agent nói).

**Hội tụ (chống loop vô hạn):** hội tụ trên **0-BLOCKER** (không phải 0-finding;
MEDIUM/LOW chỉ báo cáo) · ép **giảm đơn điệu** số blocker + fingerprint chống
dao động · **cap 3 vòng** · hết cap/dao động → escalate DEV qua tier
**`gap-cant-fix`** có sẵn ([gap_completeness_gate.py:14](../templates/claude/hooks/gap_completeness_gate.py#L14)).

**Token tối thiểu:** diff-only (không đọc cả repo, không chat) · sha-cache (skip
nếu diff-sha không đổi) · incremental re-review (vòng 2+ chỉ soi diff-của-fix +
tái kiểm blocker cũ) · 1 reviewer mặc định, model risk-scaled · skip-trivial
(diff ≤ K LOC / không feature-scope theo regex
[auto_test_runner.py:55](../templates/claude/hooks/auto_test_runner.py#L55)).

## 4. User Stories

- **Story 1 — Gate enforce review-before-done.** Là DEV, tôi muốn agent
  **không thể** báo done một feature spec-tracked khi chưa có review độc lập
  khớp diff hiện tại (ở strict=block), để blocker không lọt xuống done-claim.
- **Story 2 — Reviewer thực sự độc lập.** Là DEV, tôi muốn reviewer chỉ thấy
  artifact (diff+spec+evals+invariants) do CLI lắp, không thấy reasoning của
  implementer, để nó bắt được blind-spot mà review cùng-context bỏ sót.
- **Story 3 — Vòng review→fix luôn kết thúc.** Là DEV, tôi muốn vòng lặp hội tụ
  trên 0-BLOCKER hoặc escalate cho tôi sau ≤3 vòng, không bao giờ loop vô hạn
  (fix→sinh bug→fix).
- **Story 4 — Tốn ít token.** Là DEV, tôi muốn review độc lập chỉ chạy khi cần
  (done-boundary, diff không trivial, sha đổi) và chỉ soi diff, để chi phí token
  tối thiểu.

## 5. Implementation Decisions

- **ID-1 Trigger = done-boundary**, không per-edit. Điều kiện: tasks.md hết
  pending + `/verify` pass + agent sắp claim done.
- **ID-2 Independence by construction:** packet do CLI lắp; reviewer được lệnh
  "review only from packet". *Residual (trung thực):* agent vẫn có thể nhét thêm
  context vào prompt Task → **không chặn tuyệt đối**; enforce **mức tối thiểu
  verify-được** (sub-agent chạy thật + packet-sha tiêu thụ + artifact sha-match).
  Đây là defense-in-depth, không phải tường thép (giống `git_guardrails`).
- **ID-3 Gate verify, không tin prose** (3-lớp bằng chứng ở §2 Lớp 3). WARN
  mặc định; strict=block qua `enforce_mode`/`AGENT_TOOLKIT_STRICT`.
- **ID-4 Convergence:** 0-BLOCKER · giảm đơn điệu + finding-fingerprint · cap 3 ·
  escalate `gap-cant-fix`. **Hard-cap + `stop_hook_active`** bám
  [evidence_audit.py:33-41](../templates/claude/hooks/evidence_audit.py#L33);
  invariant cấm strip cap (C4).
- **ID-5 Token:** diff-only · sha-cache · incremental re-review (diff-của-fix +
  tái kiểm blocker cũ) · 1 reviewer/risk-scaled model · skip-trivial.
- **ID-6 Feedback:** BLOCKER xác nhận → bơm vào `.open_gaps.json` → tái dùng
  `gap_completeness_gate` để ép fix (không xây tracker mới).
- **ID-7 (was OQ-2) Skip-trivial K = 200 LOC**, reuse `loc_delta_tracker`
  `per_turn_added_warn` (loc_delta_tracker.py:42); configurable trong
  `independent_review.json`.
- **ID-8 (was OQ-3) "Spec-tracked" detection:** reuse
  `post_edit_verify_gate._file_in_implementing_spec` — file được spec status
  `implementing`/`gaps-found` tham chiếu (post_edit_verify_gate.py:143). Không
  tự định nghĩa lại.
- **ID-9 (was OQ-4) diff-sha scope = feature-scope file-set** (đúng tập review
  phủ), KHÔNG toàn cây → sửa file ngoài scope không false-invalidate artifact.
- **ID-10 (was OQ-5) Reviewer model MVP = kế thừa/default**; risk-scaled
  override = Phase 2 (Out-of-scope).
- **ID-11 (was OQ-6) Escalate theo mode của gate:** WARN → cảnh báo + cho done;
  strict(block) → chặn done tới khi DEV xử (`gap-cant-fix` surfaced). Mirror
  `enforce_mode` per_hook.
- **ID-12 (was OQ-7) Verify packet tiêu thụ bằng sha literal:** packet-sha phải
  xuất hiện trong first user-msg của sub-agent transcript (mạnh hơn match path;
  bám trust-model `pass_contract` fingerprint).
- **ID-13 (was OQ-1) Expose dual-path:** (1) **Gate auto-drive** —
  `independent_review_gate` (Stop) chặn done-claim ở done-boundary → AGENT tự
  spawn reviewer qua skill để qua gate; DEV KHÔNG gõ gì trong flow thường.
  (2) **Lệnh riêng `/review-independent` = MANUAL on-demand** (review sớm /
  re-review / trên PR). Hai path dùng chung skill `independent-review`.

### Gap-fix (đóng 4 BLOCKER trước /implement)

- **ID-14 (fix B1 — độc lập soft) Hạ kỳ vọng + verify packet-purity:** đổi
  thuật ngữ toàn spec "độc lập (tuyệt đối)" → **"fresh-context review"**
  (trung thực: KHÔNG chặn được agent nhét thêm context vào prompt Task). Verify
  tăng cường ở gate: first-user-msg của sub-agent phải (i) chứa packet-sha VÀ
  (ii) kích thước ≈ packet (≤ 1.2× packet bytes). Lệch lớn ⇒ nghi đã chèn
  reasoning ⇒ verdict `tainted-review`, KHÔNG tính review hợp lệ. → cập nhật T6.
- **ID-15 (fix B2 — cùng-model blind-spot) Prompt đối kháng BẮT BUỘC:** skill
  ép reviewer prompt = "mặc định NGHI NGỜ, cố BÁC BỎ từng hunk diff, KHÔNG giả
  định implementer đúng". Bù một phần blind-spot khi cùng model. Model-diversity
  vẫn Phase 2. → cập nhật T3.
- **ID-16 (fix B3 — done-detect brittle) Trigger theo STATE, không theo prose:**
  done-boundary xác định bằng tín hiệu CẤU TRÚC — spec `status: verified` +
  tasks.md hết pending + có diff feature-scope — KHÔNG regex `done/xong/ready`
  trên text (tránh đúng họ lỗi extraction R8 / 2 gate inert). → cập nhật ID-1, T5.
- **ID-17 (fix B4 — jam mới) Jam-escape BẮT BUỘC:** gate mang
  `stop_hook_active` + hard-cap (bám `evidence_audit.py:35,41`); sau N block
  liên tiếp HOẶC reviewer fail/timeout → **degrade WARN + escalate DEV**, không
  bao giờ jam vĩnh viễn. Invariant (T10) phủ luôn cap này. → cập nhật T6, T7.

### Gap-fix (đóng 5 MEDIUM)

- **ID-18 (fix M1 — diff-sha thô) Hash bao trùm contract + normalize:** review
  sha = hash(diff feature-scope **+ spec + acceptance_evals + invariants liên
  quan**), normalize bỏ comment/whitespace-only trước hash. ⇒ sửa spec/evals
  làm stale đúng (re-review); sửa comment không gây re-review thừa. → T1.
- **ID-19 (fix M2 — token phình) Cap + prune packet:** packet **prune sẵn** file
  liên quan (import/inherit của diff, static-light) thay vì để reviewer tự đào;
  config `max_reads_per_round` (default 10); vượt cap → reviewer kết luận với
  context hiện có + note `context-limited`. → T1/T3.
- **ID-20 (fix M3 — monotonic deadlock) Phân biệt regression vs issue-mới-hợp-lệ:**
  non-converge CHỈ khi (a) finding tái xuất (fingerprint cũ) HOẶC (b) blocker mới
  nằm TRONG vùng diff của bản fix (= regression). Blocker mới ở vùng khác (đào
  sâu hợp lệ) KHÔNG tính non-converge nhưng có **trần tuyệt đối** (vd 5 vòng) để
  vẫn kết thúc. → T7.
- **ID-21 (fix M4 — finding ảo) Verify mỗi BLOCKER trước khi ép fix:** trước khi
  bơm vào `.open_gaps.json`, reviewer (hoặc 1 lớp phản biện) phải chứng minh
  blocker **reproduce/chứng minh được bằng file:line + cơ chế cụ thể**. Không
  chứng minh được → **hạ MEDIUM** (báo cáo, KHÔNG ép fix). Tránh "feature bắt
  bug lại tạo bug". → T6/T8.
- **ID-22 (fix M5 — Stop-timeout/orchestration) Cap đọc + thứ tự rõ:** gate đọc
  **tail transcript có cap 256KiB** (như `_last_tool_use_is_pending`); timeout
  riêng cho gate (15s). Thứ tự: reviewer ghi artifact → agent-loop kết thúc lượt
  → Stop-hook **chỉ ĐỌC** artifact có sẵn (hook KHÔNG đợi sub-agent; spawn xảy ra
  ở vòng trước). → T6/T9.
