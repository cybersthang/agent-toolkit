# v0.34.0 — Enforcement Hardening Phase 2

## 1. Problem / Why

The adversarial deep-dive scored the **enforcement layer** at ~2.1 (default) / 2.6
(strict). v0.33 closed **① under strict** (un-forgeable result-inspection) + **⑤**,
but a DEFAULT `setup.py init` install still gets the old soft behaviour. The score
is held under 4 by **three architectural root causes** (code-grounded):

| Root cause | Evidence (path:line) | Effect |
|---|---|---|
| **N1 — default WARN/OFF** | `verify_lint.py:317` `default="warn"` | even ① (un-forgeable) only blocks under strict → default install unchanged |
| **N2 — gate reads agent-authored state / shape only** | ② `implement_notes_gate.py:302` `.exists()`-only; ④ `independent_review_gate.py:177` reads agent-writable `subagents/` (deny=0) | agent fabricates the state that satisfies the gate |
| **N3 — verification per-TURN, not per-finding** | ③ `_audit/claim_audit.py:66-77` `has_evidence` True on ANY one tool in the whole turn | a 40-finding `/review` passes with one `Read` |

**Why 4.0 is reachable now (it wasn't at v0.27):** v0.27 paralysis came from
**broad-blocking JUDGMENT gates** (gap/scope: "đủ chưa?" — fuzzy, frequent, no
convergence). We now have two things v0.27 lacked: (1) **CRISP un-forgeable
triggers** (① result-inspection proven, ② 0-byte, ③ proof-exists, ④ forge) that are
binary not fuzzy; (2) **convergence machinery** (`independent_review_gate` jam-escape
`block_streak≥3 → degrade+escalate`; `evidence_audit` `_RECURSION_HARD_CAP=3`) that
guarantees no infinite loop. → Block-default on CRISP gates + convergence = 4.0
WITHOUT paralysis.

## 2. Goals / Non-goals
**Goals:** raise DEFAULT enforcement (①②③④) to ~4.0 via crisp block-default +
convergence; keep judgment gates (gap/scope/post_edit) at warn; ship the de-risk
(R5) so a false-positive can never paralyze.
**Non-goals:** broad block of judgment gates (the v0.27 trap); cryptographic
anchor for ④ (→ v0.35); per-version Odoo CI matrix.

## 3. User Stories
- **US1 (①)** — a DEFAULT install must block a feature-scope done-claim with no acceptance_evals / no real-data probe.
- **US2 (②)** — empty/section-missing sidecar must not satisfy the gate.
- **US3 (③)** — every `/review` Proof line is per-finding validated (PATH exists + a tool touched that path this turn; line-level is NOT mechanically verifiable).
- **US4 (④)** — consumption is not forgeable by an agent Write; verdict is reviewer-authored.
- **US5 (F1.5)** — a real run via `poetry/uv/xvfb-run/timeout pytest` counts as a probe.
- **US6 (R5 de-risk)** — a false-positive block must auto-degrade (never deadlock) AND a trigger flips warn→block-default only after telemetry shows FP≈0.

## 5. R5 — De-risk (the mechanism that makes block-default SAFE)
*All reuse infrastructure verified to exist.*
- **R5.1 Convergence-cap (trigger-class-aware — rev2):** *(fix: blanket degrade-to-warn rewards refusal + caps crisp enforcement below 4.0).* **Crisp/cheaply-satisfiable (0-byte, no-evals, fake-proof):** `block_streak ≥ K → escalate-and-HOLD` — require the bypass token to proceed, do NOT auto-allow (a K-streak on a trivial fix = refusal, not deadlock). **Legitimately-unsatisfiable (reviewer-unavailable):** `block_streak ≥ K → degrade-to-warn + escalate` (the proven `independent_review_gate.py:295` path). Either way: **no infinite loop, and crisp triggers stay enforced.**
- **R5.2 Shadow-block + sampled-FP-review (the flip gate — rev2):** *(fixes the chicken-egg: `bypass_rate_alarm` measures BYPASS-rate, NOT false-positive — a WARN that would-wrongly-block is indistinguishable from a correct WARN in `.hook_fire_log.json`).* At WARN, each gate emits a structured **`would-block`** fire-event (trigger + spec + expected satisfying-action). **FP-rate = R5.3 SAMPLING those `would-block` events** (independent-review / human judges "would this block LEGIT work?"), NOT derived from bypass-rate. Flip warn→block-default per-trigger ONLY when sampled would-block FP≈0. `bypass_rate_alarm` is a SECONDARY post-flip signal, not the FP gate.
- **R5.3 Adversarial FP-review:** per gate, an independent fresh-context agent hunts for LEGIT work the trigger wrongly blocks (not just bypasses); fix before flip.
- **R5.4 Single-shot escape (per-gate — rev2 correction):** a logged escape token per block-default gate. NOTE: `implement_notes_gate` already has `implement-notes: skip`; `review_proof_gate` adds one (T7); **`verify_lint` has NONE today → T3b adds one** (else crisp escalate-and-HOLD deadlocks). Universal fallback: `enforce_mode` strict-off (R1 opt-out).
- **R5.5 Feature-scope only:** triggers fire only on `coverage_config.json` feature globs (limited blast radius).

## 5b. Risks (enumerated — R5 above are the mitigations)
- **R0 — v0.27 paralysis (meta-risk):** broad block of judgment gates. → ONLY crisp gates block; gap/scope/post_edit STAY warn (§Goals) + R5.1 convergence.
- **R1 — friction (Q1=a block-default ALL installs):** upgrade-shock. → flip post-FP≈0 (R5.2) + migration note + opt-out + R5.1.
- **R2 — review_proof_gate false-block** a legit finding citing an untouched path. → warn-first + R5.3 FP-review + R5.4 escape.
- **R3 — F4.2 reviewer↔gate contract change** (biggest surface). → ship warn-capable + own adversarial round; F4.1 alone already → ~3.5.
- **R4 — F2.2 SD-match snapshot-dependent + FUZZY heuristic.** → keep at WARN (don't flip without R5.3); degrade to warn when no snapshot.
- **R5(risk) — dogfood self-block during implement:** editing `templates/claude/hooks/*.py` is ITSELF feature-scope under the gates being hardened → the implementer can self-block under strict. → `enforce_mode` warn for the toolkit's own dev loop (or scope-exempt `templates/`) during T1-T13.
- **R6(risk) — `subagents/` write-provenance UNVERIFIED (HIGH-2):** F4.1's "harness writes it" is an assumption → confirm empirically before F4.1 ships + keep F4.2's verdict path outside the deny-glob.
