# Retrospective spec — agent-toolkit autonomy chain v0.6.0 + v0.6.1

## 1. Problem

Original DEV ask: "AGENT should auto-cover the 9 manual interventions DEV had to make in the Hotpot Dashboard session" — `tự run test đi`, `dùng MCP Playwright đi`, paste-URL+creds, `kill rồi chạy lại`, recipe-driven probes, `fix đi`, `fix hết đi`, `có blocker hay GAP không`, `probe-skip` spam.

## 5. Implementation Decisions

D1. **Schema v2 backward-compat via `schema_version` field** — old v1 probes default to `auto_run: false`, `recipe_drift_tolerance: medium`.

D2. **MCP-call bridge prefers direct-spawn** — `TOOLKIT_MCP_CLIENT=claude_cli` opt-in only; Node + claude-cli flag not assumed.

D3. **Auto-fire default OFF** — `auto_run: false` on every probe post-migration; `auto_start_if_dead: false` on daemon. DEV explicitly opts in to avoid surprise execution.

D4. **STRONG/WEAK pattern split** — bare exception names need `File "...", line N | Traceback | caret` within ±200 chars window.

D5. **Plugin interface for diagnose strategies** — drop `.py` in `.codex/gap_fix_diagnose/` with `matches()` + `diagnose()` signature; engine auto-discovers.

D6. **Project-customized debug.json patterns treated STRONG** — DEV explicitly opted in → no context check (back-compat for v1 configs).
