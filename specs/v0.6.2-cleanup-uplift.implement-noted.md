## 1. Scope deviations (6 items)

### SD-1: Expanded `DEFAULT_FEATURE_GLOBS` to support BOTH flat + nested layouts

- Type: outside-spec
- Lý do: Spec eval `g1-warn-on-feature-edit-without-spec` expected
  match on `models/thing.py` (flat layout) but original globs only
  had `**/models/**/*.py` which fnmatch translates to require ≥1
  subdirectory after `models/`. Test failed initial run; expanded
  defaults to include both `**/models/*.py` and `**/models/**/*.py`
  (plus same pair for controllers/wizards/jobs/app/controllers/etc).
- Alternatives considered: change test to use nested file path
  (`models/sub/thing.py`); change globs ONLY (chosen — more robust
  for real projects).
- File(s) affected: `templates/claude/hooks/spec_first_guard.py:62-93`
- Spec linkage: g6-config-driven-globs (defaults updated)
- Confidence: high
- DEV review urgency: low

### SD-2: Dropped `^` anchor in test src_regex pattern

- Type: diverged-from-request
- Lý do: `auto_test_runner._find_mapping()` uses `re.search` (not
  `re.match`); my test's `^addons/...` anchored regex failed against
  absolute file paths because path starts with `D:/Temp/...`. Spec
  eval `c9-hook-integration-tests` only required ≥3 tests pass, not
  specific regex shape.
- Alternatives considered: change auto_test_runner to re.match
  (rejected: more invasive, broader impact).
- File(s) affected: `tests/test_hooks_integration.py:122,176` (2
  occurrences via replace_all)
- Spec linkage: c9-hook-integration-tests
- Confidence: medium
- DEV review urgency: medium (anchor drop might mask future bugs;
  consider documenting in auto_test_runner that src_regex is
  unanchored)

### SD-3: Added `symbolic-ref` fallback in `_resolve_branch`

- Type: outside-spec
- Lý do: Test `TestWarnOnFeatureEdit` failed because fresh
  `git init` + `git checkout -B feature-foo` (no initial commit yet)
  causes `git rev-parse --abbrev-ref HEAD` to return "HEAD" with
  rc=128. Real-world: this happens on first push of feature branch
  before commits land. Spec didn't anticipate.
- Alternatives considered: require tests to make initial commit
  (rejected: would mask real production scenario where unborn-branch
  Edits do happen).
- File(s) affected: `templates/claude/hooks/spec_first_guard.py:111-145`
- Spec linkage: g1-warn-on-feature-edit (made more robust)
- Confidence: high
- DEV review urgency: low
