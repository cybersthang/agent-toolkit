# Spec — v0.6.2 cleanup + uplift sprint

## 1. Problem

State sau v0.6.0+v0.6.1: score 8.42/10 với gaps cụ thể:
- Version metadata stale (0.5.0 vs CHANGELOG 0.6.x).
- `.coverage` binary committed despite gitignore.
- Duplicate spec paths (specs/ + .agent-toolkit/specs/).
- mcp_call success path untested.
- Django/Rails recipe patterns mới ở mức seed (~25 LOC).
- gap_fix_cycle engine chưa run E2E.
- 3 PostToolUse hooks chưa test integration với MCP.
- verify_report.md evidence links ở mức name-only.

## 2. Solution

Sprint bao gồm 10 cleanup + uplift items (c1-c10), không thêm feature
mới. Mục tiêu: state pristine cho v0.7.x feature work (spec_first_guard
sẽ ship parallel ở `specs/v0.7.0-spec-first-guard.md`).

## 3. Affected modules

- `lib/installer.py` — version bump
- `.gitignore` already complete; need `git rm --cached`
- `specs/v0.6.0-autonomy-chain.*` — consolidate canonical
- `templates/codex/tools/detect_retrospective_spec.py` — new tool
- `templates/agent_toolkit/decision-log.md` — new ADR seed
- `tests/test_*.py` — 4 new test files (version, retrospective, mcp_call success, recipe patterns, hooks integration)
- `templates/codex/recipe_patterns/{django,rails}_triggers.json` — expand
- `specs/v0.6.2-gap-fix-cycle-trace.md` — new evidence doc
- `specs/v0.6.0-autonomy-chain.verify_report.md` — evidence tightening

## 4. Implementation Decisions

D1. **Version scheme**: bump to 0.6.2 (cleanup release).
v0.7.0 reserved for spec_first_guard feature.

D2. **Canonical spec path = `specs/`** — `.agent-toolkit/specs/` copy was
dogfood-only; delete after sprint completes.

D3. **`detect_retrospective_spec.py` heuristic** — compare `git log
--follow` first-commit timestamp của spec.md vs first-commit timestamp
của bất kỳ feature-scope file landed dưới `module` field. Spec sau →
flag retrospective.

D4. **Recipe expansion strategy** — Django + Rails patterns dựa trên
common test framework idioms + ORM lifecycle hooks. KHÔNG hardcode
project-specific paths (e.g. không có `myapp/models.py` literal — chỉ
generic patterns).

D5. **Integration test pattern** — fake MCP server fixture
(subprocess listening on Unix domain socket / Windows named pipe / TCP
loopback) returning canned JSON-RPC responses. Hook sends real payload;
test asserts payload shape.

D6. **gap_fix_cycle trace** — inject `assert False` vào 1 probe falsify
script copy (không touch original), run engine, capture stdout +
written patch artifacts to `specs/v0.6.2-gap-fix-cycle-trace.md`.
