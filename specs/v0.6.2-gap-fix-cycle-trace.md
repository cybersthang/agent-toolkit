## What was deliberately NOT exercised

- Iteration > 1 (max-iter loop continuation with stale state).
- Real timing_perturb / playwright runner via falsify (demo uses
  log_assertion shape but minimal runner content).
- Full success path where re-run returns rc=0 → engine exits with
  `proven` status. Demo stops at `error` rerun.

These are explicit gaps; real-world workflows produce them organically.
