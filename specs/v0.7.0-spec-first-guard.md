# Spec — v0.7.0 spec_first_guard hook

## 1. Problem

Vibe-flow Phase 1 ("Plan-first") là contract của toolkit nhưng hiện
chưa có enforcement mechanism. v0.6.0 patches đã ship với
retrospective spec (`specs/v0.6.0-autonomy-chain.md` có
`retrospective: true`) — chính dogfood event này expose gap.

Hậu quả khi không có guard:
- DEV / AGENT có thể Edit feature-scope file mà chưa định nghĩa
  `acceptance_evals` → verify_report.md không có rubric mechanical.
- Retrospective spec dễ trở thành "rationalization" thay vì design.
- Tăng nguy cơ scope creep / drift giữa intent ban đầu và code landed.

## 2. Solution

PreToolUse hook `spec_first_guard.py`:
- Trigger trên Edit/Write/MultiEdit/NotebookEdit.
- Resolve current git branch.
- Skip nếu branch ∈ {main, master, trunk}.
- Skip nếu file_path match test-glob (tests/**, *_test.py, *.test.js).
- Skip nếu file_path KHÔNG match `feature_scope_globs` (config-driven).
- Look up spec: `.agent-toolkit/specs/**/<branch-slug>.md` (recursive).
- Nếu spec found + `acceptance_evals:` non-empty → silent.
- Nếu spec missing / empty evals → WARN (stderr line, không block).
- Bypass: `spec-first-guard: skip <reason>` trong tool input field.
- Fail-open: mọi error → exit 0 silent.

Hook KHÔNG block tool call → DEV workflow không bị cản. Mục tiêu là
**nudge** + record event vào `.agent-toolkit/.spec_first_guard_log.json`
để gap_status có thể đếm.

## 5. Implementation Decisions

D1. **Warn-only, not block** — Toolkit's contract value is "evidence-first",
not "spec-police". Block-mode rejected because:
- Hotfix / typo-fix shouldn't require spec.
- Spec viết được trong Edit thứ 2 (sau khi nhìn code 1 lần) cũng OK.
- Block-mode = DEV phải bypass mỗi commit → noise.

D2. **Branch resolution via `git rev-parse --abbrev-ref HEAD`** —
fallback empty string if git unavailable (rare case: not a repo).
Empty branch ↔ treat as trunk → no warn.

D3. **Spec lookup via `rglob`** — consistent với
`verify_lint.py:46` + `verify_nudge.py` + `analyze_halt_gate.py` đã
rglob. Branch-scoped layout `.agent-toolkit/specs/<branch>/<slug>.md`
cũng được pick up.

D4. **Slug match: spec stem == branch name** — đơn giản hóa lookup.
Fallback: any spec with `branch: <name>` field trong frontmatter cũng
được count. Cover case branch=feature/x → spec slug=feature-x.

D5. **Config-driven globs** — default fallback:
```python
DEFAULT_FEATURE_GLOBS = [
    "**/models/**/*.py", "**/controllers/**/*.py",
    "**/wizards/**/*.py", "**/jobs/**/*.py",
    "**/views/*.xml", "**/security/*.csv",
]
```
Đây là Odoo-flavored seed. Project override qua
`coverage_config.json` → `feature_scope_globs: [...]`. Django dev sẽ
override thành `["**/views.py", "**/models.py", ...]`. Public PR welcome
cho mỗi stack.

D6. **Skip test files via dedicated test-glob** — separate from
feature_scope_globs để tránh false positive. Default test-glob:
```python
DEFAULT_TEST_GLOBS = [
    "**/tests/**", "**/test_*.py", "**/*_test.py",
    "**/*.test.js", "**/*.spec.js", "**/spec/**",
]
```

D7. **Log to JSON for gap_status integration** —
`.agent-toolkit/.spec_first_guard_log.json` ring buffer (last 50
events). gap_status có thể đếm `warn_count` để show ở next_action.
