# Spec — v0.7.1 implement-notes artifact

## 1. Problem

Toolkit có nhiều disclosure layer cho UNCERTAINTY (`[assumption]`,
`probe-skip`, `progress-skip`) và INTENT (spec + clarification-gate
ASSUMPTIONS), nhưng **CHƯA CÓ** cơ chế nào surface "AGENT đã ra quyết
định gì ngoài spec lúc implement". Code show WHAT, không show WHY.
Git commit messages cũng thường ngắn + post-hoc.

Hệ quả: DEV mở PR thấy 5000 LOC diff, không biết AGENT đã pick choice
A vs B vì lý do gì cụ thể. Failure mode: scope-creep detect quá muộn
(sau merge); silent decisions trở thành tech-debt.

## 2. Solution

Per-spec sidecar artifact `<slug>.implement-noted.md` capture 4 thứ:
1. **Scope deviations** — quyết định AGENT outside spec (merged "decisions outside spec" + "changes from request" của DEV's original proposal vì 2 cái overlap quá nhiều).
2. **In-transcript trade-offs** — STRICT: chỉ items có evidence trong transcript turn (cite turn id / timestamp). Cấm post-hoc reconstruction để giảm hallucination risk.
3. **Open follow-ups** — items AGENT noticed nhưng defer cho DEV.
4. **Confidence summary** — AGENT tự rate high/medium/low per decision; flag low-confidence items cho DEV review urgent.

3-section breakdown gốc của DEV ("decisions outside spec / changes from request / trade-offs considered") được revise sau Raw Opus 4.7 Max High analysis: category 2 merge vào 1 vì overlap; category 3 thêm strict cite-required constraint; thêm category 4 confidence để giảm honor-system fragility.

## 4. Implementation Decisions

D1. **Merge categories 1+2** từ DEV's proposal → "Scope deviations". Lý do: DEV's request → spec (qua /clarify) → implementation. Divergence between request và spec đã handle bằng /clarify; còn lại spec↔impl = scope deviations. 2-category 1+2 chỉ là labels khác cho cùng concept.

D2. **Trade-offs strict cite-required**: AGENT không reliable introspect decision process. Chỉ liệt kê trade-offs **có evidence trong transcript** (turn id / message id / timestamp). Reduces hallucination risk.

D3. **Hook warn-only** (R3, không R4): R4 hard enforcement defer cho đến khi R1+R2 demonstrate value. Honor-system layer chấp nhận initially.

D4. **Per-spec sibling** path: `.agent-toolkit/specs/<branch>/<slug>.implement-noted.md`. Consistent với existing `<slug>.verify_report.md` pattern.

D5. **Bypass marker**: `implement-notes: skip <reason>` token. Single-shot. Same pattern như `spec-first-guard: skip` / `debug-sentry: skip`.

D6. **Public-project safe**: no hardcoded stack name / path. Schema generic. Skill works for any feature_kind.
