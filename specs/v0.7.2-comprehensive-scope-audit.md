# Spec — v0.7.2 Comprehensive scope audit

## 1. Problem

Implement-noted artifact (v0.7.0 ship) là output-only. AGENT có thể:
- Edit files ngoài declared spec scope (file-level creep).
- Edit files đúng scope nhưng làm việc unrelated (semantic creep).
- Fabricate SD entries (hallucinated decisions).
- Omit SD entries (missing decisions).

DEV constraint từ turn trước: workflow phải giữ 2 touch points (Plan + Verify). Không thể yêu cầu DEV đọc implement-noted manual để phát hiện 4 failure modes trên.

Cần **mechanical** safety net AGENT auto-run trước /verify — không AGENT self-judgment subjective.

## 2. Solution — 4-coverage matrix

| # | Failure mode | Mechanism |
|---|---|---|
| 1 | File-level scope creep | Layer 5: files modified vs spec.affected_modules + snapshot diff |
| 2 | Semantic creep inside scope | Diff-hunk annotation: every hunk MUST tag eval id, SD ref, hoặc bypass |
| 3 | Hallucinated SD | Validator: SD path/line/eval-id phải tồn tại |
| 4 | Missing SD | Cross-check transcript Edit count vs SD count + eval target match |

Coverage 5 (stale F-N), 6 (cross-feature mining), 7 (compute waste) defer v0.8+.

## 3. AGENT Phase 5 orchestration (DEV-unaware)

```
/implement <slug> Phase 5 sequence (AGENT auto):
  5.0  Emit <slug>.implement-noted.md (v0.7.0 existing)
  5.1  Run implement_noted_validator → flag hallucinated SD/T/F
  5.2  Run missing_sd_detector → flag undeclared Edits
  5.3  Run diff_hunk_annotator → AGENT tag each hunk; validator asserts coverage
  5.4  Run verify_lint scope check → file-level scope creep
  → Clean? Proceed /verify
  → Issues? Iter 2: auto-fix or block with diagnostic
```

DEV touch unchanged: `/plan`, `/clarify` answer, `/verify`, đọc verify_report.

## 4. Affected modules

See frontmatter `affected_modules`. Layer 5 enforce: edits outside list → block.

## 5. Implementation Decisions

D1. **Snapshot mechanism over git operations**: AGENT không commit; "revert" / "diff vs baseline" cần snapshot directory primitive. Storage cost ~5-50KB per active feature, negligible.

D2. **Warn-only initial** trên Layer 5 + diff annotation block: false-positive risk khi mới ship; upgrade hard sau khi 3-5 features chạy thử. Toggle qua `.agent-toolkit/scope_audit.json` `enforce: warn | block`.

D3. **Diff annotation cite-required strict**: each hunk MUST have annotation. AGENT generate annotations TỰ ĐỘNG ở Phase 5.3 prompt; validator only checks completeness.

D4. **Spec retrofit policy**: backfill affected_modules cho 5 existing specs từ git log. Grandfather option deferred — risk false-positive trên legacy.

D5. **Public-project safe**: zero hardcoded stack/project names. Defaults configurable qua `.agent-toolkit/coverage_config.json`.

D6. **Bypass marker family extension**:
- `scope-creep-allowed: <file> <reason>` — file-level exempt.
- `untagged-hunk-allowed: <line-range> <reason>` — hunk-level exempt.
