## 1. Scope deviations (3 items)

### SD-1: Annotator regex needed widening for SD title line text

- Type: outside-spec
- Lý do: Initial `_extract_sd_file_refs` heading regex
  `^#{2,4}\s*SD-(\d+)\s*[:\-](?P<body>...)` failed because body pattern
  expects newline immediately after `:`, but actual SD heading has
  ` stuff` title text on same line. Test `test_auto_tag_matches_sd_file_ref`
  caught this. Fixed by adding `\s*[^\n]*` after the colon to consume
  remaining title content.
- Alternatives considered: keep narrow regex + change SD format
  (rejected: would invalidate every existing implement-noted file).
- File(s) affected: `templates/codex/tools/diff_hunk_annotator.py:50-70`
- Spec linkage: o3-annotator-auto-tag
- Confidence: high
- DEV review urgency: low

### SD-2: Orchestrator runs sub-tools via subprocess, not import

- Type: outside-spec
- Lý do: Spec didn't specify invocation mechanism. Chose subprocess for:
  isolation (each tool's import-time side effects don't leak); easier
  timeout enforcement; matches existing pattern (verify_lint.py invokes
  lint_verify_report.py via subprocess).
- Alternatives considered: importlib import + call into library API
  (rejected: tools have CLI-only `main` entry points; refactor large).
- File(s) affected: `templates/claude/hooks/implement_orchestrator.py:131-149`
- Spec linkage: o1-master-orchestrator-hook
- Confidence: high
- DEV review urgency: low

### SD-3: 60s cache TTL hardcoded (not config-driven)

- Type: outside-spec
- Lý do: Idempotency cache to avoid re-running chain on every Stop event
  within same response. 60s heuristic — sufficient for typical
  conversation pace, prevents loop. Could be config field but no DEV ask.
- Alternatives considered: cache via session-id (rejected: Claude Code
  doesn't expose session id in Stop envelope reliably).
- File(s) affected: `templates/claude/hooks/implement_orchestrator.py:30`
- Spec linkage: o1-master-orchestrator-hook
- Confidence: medium
- DEV review urgency: medium (DEV may want config knob)
