# Spec — v0.7.3 Orchestrator + E2E + Auto-tag

## 1. Problem

Sprint v0.7.2 ship 4-coverage scope audit nhưng self-review phát hiện
3 gaps trong "xuyên suốt":
- **Gap 1**: No master orchestrator — Phase 5.1-5.4 mô tả trong SKILL.md
  nhưng không có hook chain tự động. AGENT phải voluntarily invoke
  từng tool. Honor-system tại orchestrator layer.
- **Gap 3**: 41 component tests, 0 integration tests for full chain.
  Cross-component data flow proven only at component-level.
- **Gap 4**: Diff annotator emits template với placeholder, AGENT phải
  fill mọi `tag:` field. Burden cao + dễ skip.

Hệ quả: chain Plan→Implement→Phase 5.1-5.4→Verify chỉ đạt ~70% xuyên
suốt (per Raw Opus 4.7 Max High self-evaluation).

## 2. Solution

3 components đóng 3 gaps:

### O1 — Master orchestrator Stop hook (Gap 1)

`templates/claude/hooks/implement_orchestrator.py`:
- Trigger: Stop event với done-claim OR Verify Report marker.
- Resolve current branch → slug → spec.
- Skip if spec lacks affected_modules (grandfather).
- Auto-invoke chain:
  1. `implement_noted_validator.py` → check hallucinated SD.
  2. `missing_sd_detector.py` → check missing SD.
  3. `diff_hunk_annotator.py --write` → emit annotation template.
  4. `diff_annotation_validator.py` → check untagged hunks.
- Aggregate verdicts → emit single additionalContext to AGENT.
- Idempotent: cache verdict trong `.agent-toolkit/.orchestrator_state.json`,
  skip if recent run within 60s.
- Fail-open: any tool error → exit 0 silent.

### O3 — Annotator auto-tag inference (Gap 4)

Modify `diff_hunk_annotator.py`:
- Load spec.acceptance_evals → extract probe.args.target files.
- Load impl-noted SD-N entries → extract file refs.
- For each hunk:
  - File matches eval target → auto-tag với eval id.
  - File matches SD-N ref → auto-tag với SD-N reference.
  - Otherwise → leave `<FILL ...>` placeholder.
- Frontmatter: `total_auto_tagged: N` field added.

AGENT only fills residual hunks (typically ≤20% of total).

### O2 — E2E integration test (Gap 3)

`tests/test_v073_e2e_chain.py`:
- Provision tmp project với spec + affected_modules.
- Sequential simulation:
  1. PreToolUse envelope → snapshot hook captures pre-state.
  2. Multiple Edit envelopes (mix in-scope + out-of-scope).
  3. AGENT emits implement-noted.md sidecar.
  4. Stop envelope với done-claim → orchestrator fires.
  5. Assert: validator verdict, detector verdict, annotator template
     emitted, scope-check verdict.
- ≥3 scenarios:
  - Clean flow: all in scope → LEAN.
  - Out-of-scope Edit → detector flags + verify_lint_scope warns.
  - Missing SD → detector flags missing.

## 4. Implementation Decisions

D1. **Orchestrator as Stop hook, not skill**: skill = AGENT discipline;
hook = mechanical fire. Choice = hook để guarantee execution.

D2. **Aggregator pattern**: orchestrator doesn't re-implement logic, just
chain existing tools via subprocess. Easier to test + extend.

D3. **Auto-tag heuristic**: simple file-path match. AGENT can override by
editing the auto-tagged value. Conservative — doesn't auto-tag if 
multiple eval targets match same file (ambiguous).

D4. **E2E test simulates envelopes**, not actual Claude Code dispatcher.
Same constraint as existing smoke tests. Live dispatcher exercise still
DEV-manual T1 backlog.

D5. **Bypass marker family extended**:
- `orchestrator-skip: <reason>` — skip entire orchestrator chain
  single-shot.

D6. **Public-project safe**: no hardcoded project names.
