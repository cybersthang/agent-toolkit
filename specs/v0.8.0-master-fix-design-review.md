## Section 3 — Interaction matrix (critical)

### 3.A Stop hook cascade — primary concern

Claude Code Stop hook behavior `[assumption]` based on observed events:
**First hook to emit `decision: "block"` short-circuits subsequent hooks.**

**Verification status**: empirically observed in this conversation —
when `evidence_audit` blocked, AGENT did not receive subsequent Stop
hook outputs. But not yet rigorously tested with all 8 hooks.

If `[assumption]` is correct, then Stop hooks #1-4 (all blockers) can
each prevent downstream hooks #5-8 from firing.

#### Hook #1 — `evidence_audit` block conditions

Triggers `decision: block` when:
- Response contains PASS/DONE/VERIFIED claim word.
- No MCP `realdata_test` / `postgres` call in same turn.
- No `probe-skip` / `progress-skip` token.
- No `[assumption]` tag.

**This is the single biggest cascade risk**. AGENT response with "implement done" without MCP backing = ALL downstream hooks skip:
- ❌ `verify_lint` (coverage check skipped)
- ❌ `post_edit_verify_gate` (spec-tracked file check skipped)
- ❌ `debug_sentry` (traceback check skipped)
- ❌ `spec_drift_advisory` (advisory skipped)
- ❌ `implement_notes_gate` (file existence skipped)
- ❌ `implement_orchestrator` (entire Phase 5.1-5.4 chain skipped)
- ❌ `verify_lint_scope` (Layer 5 scope check skipped)

**This is the root cause of "xuyên suốt only ~66%"** post-v0.7.3.

#### Hook #2 — `verify_lint` block conditions

Triggers block when:
- Verify Report header detected in response.
- Coverage of acceptance_evals incomplete.
- Classifier spec missing Real-Data Proof section.

#### Hook #3 — `post_edit_verify_gate` block conditions

Triggers block when:
- AGENT claims "done" on a spec-tracked file without `/verify` run this session.

#### Hook #4 — `debug_sentry` block conditions

Triggers block when:
- Traceback / qualified exception detected in response.
- No `[assumption]` / `debug-troubleshoot` / `debug-sentry: skip` token.

### 3.B PreToolUse cascade

Same logic `[assumption]`: `permissionDecision: "deny"` from hook #1 or #2 prevents hooks #3-4 from firing.

- If `invariant_guard` denies → `analyze_halt_gate`, `spec_first_guard`, `implement_snapshot_hook` skip.
  - Consequence: snapshot NOT taken → Layer 5 has no baseline for THAT file → silent dysfunction for missing-SD detection.
- If `analyze_halt_gate` denies → `spec_first_guard`, `implement_snapshot_hook` skip.

### 3.C PostToolUse cascade

PostToolUse hooks DON'T block tool calls (mode=warn only). Each fires independently `[assumption]`. Timeout failure of one shouldn't affect next.

But: combined PostToolUse timeout = 5 + 35 + 8 + 5 + 600 + 300 + 90 = **1048s worst case** (~17 minutes). If any hits timeout, AGENT loses time.

### 3.D Hook ↔ artifact dependency graph (critical reads)

```
spec.md (DEV-written)
  ←─ spec_first_guard (looks up branch slug)
  ←─ implement_snapshot_hook (needs affected_modules)
  ←─ implement_notes_gate (checks file presence)
  ←─ implement_orchestrator (drives chain)
  ←─ verify_lint (acceptance_evals)
  ←─ verify_lint_scope (affected_modules)
  ←─ post_edit_verify_gate (status check)
  ←─ spec_drift_advisory (recipe vs script)

implement-noted.md (AGENT Phase 5.0)
  ←─ implement_notes_gate (existence)
  ←─ implement_orchestrator (validator + detector use)
  ←─ verify_lint_scope (SD refs for bypass)

.implement_snapshots/<slug>/_manifest.json (snapshot_hook write)
  ←─ missing_sd_detector
  ←─ verify_lint_scope
  ←─ implement_orchestrator (via subprocess to detector)

acceptance-probes.json (DEV-customized, toolkit-seeded)
  ←─ auto_run_probes
  ←─ probe_autostub
  ←─ evidence_audit
  ←─ falsify.py
  ←─ gap_status / gap_fix_cycle

.orchestrator_state.json (orchestrator self-write)
  ←─ implement_orchestrator (idempotency cache)
```

**Critical observation**: `implement_orchestrator` reads spec + impl-noted + invokes 3 subprocess tools. Если bất kỳ artifact nào missing → silent skip → orchestrator never reports issue. **Dependencies fragile.**

## Section 4 — Adversarial failure mode enumeration

### F1 — `evidence_audit` blocks → Phase 5.1-5.4 audit chain skipped (G7 from prior review)

**Scenario**: AGENT response includes "implement done" without MCP probe call in same turn.

**Trace**:
1. Stop event fires.
2. `evidence_audit` runs first.
3. Detects PASS-claim, no MCP call → `decision: block` emit.
4. Claude Code re-prompts AGENT.
5. Hooks #2-#8 ALL skip.
6. `implement_orchestrator` NEVER FIRES → Phase 5.1-5.4 audit silently bypassed.
7. AGENT re-emits with `probe-skip:` marker (per conversation pattern).
8. `evidence_audit` allows.
9. Stop chain continues through hooks #2-#8 successfully.
10. `implement_orchestrator` fires NOW → but maybe `implement-noted` already emitted in iter 1.

**Severity**: HIGH — most common failure mode in real conversations.

**Probability of occurrence**: ~50-70% based on this conversation `[assumption]` (DEV's responses triggered evidence_audit multiple times).

### F2 — `invariant_guard` denies → snapshot not taken

**Scenario**: AGENT Edits file that violates an invariant (e.g. strips `must_keep_regex` pattern).

**Trace**:
1. PreToolUse fires.
2. `invariant_guard` denies → edit rejected.
3. Hooks #2-#4 skip.
4. `implement_snapshot_hook` NEVER FIRES for this Edit.
5. AGENT fixes invariant → Edit again.
6. PreToolUse fires.
7. NOW snapshot fires.
8. BUT: snapshot captures fixed state, not original pre-invariant-violation state.

**Severity**: LOW — invariant violations rare; even if occurs, snapshot still useful for later edits.

### F3 — Spec lacks `affected_modules` → entire Layer 5 silently skipped

**Scenario**: DEV writes spec without `affected_modules` field (legacy or oversight).

**Trace**:
1. `implement_snapshot_hook` reads spec → no affected_modules → silent skip.
2. No snapshot taken.
3. AGENT does Edits.
4. AGENT claims done.
5. `implement_orchestrator` reads spec → no affected_modules → silent skip.
6. `verify_lint_scope` reads spec → no affected_modules → can't compute scope creep → silent skip.

**Severity**: HIGH — entire Layer 5 mechanism silently dysfunctional, DEV doesn't know.

### F4 — `implement-noted.md` not emitted → orchestrator skips

**Scenario**: AGENT forgets to emit implement-noted (or uses `implement-notes: skip` bypass).

**Trace**:
1. AGENT claims done.
2. `implement_notes_gate` warns "file missing" (non-blocking).
3. `implement_orchestrator` reads → impl-noted missing → silent skip.
4. Phase 5.1-5.4 audit never runs.

**Severity**: MEDIUM — gate hook warns but orchestrator goes silent. Inconsistent feedback to AGENT.

### F5 — Multi-turn iteration → orchestrator cache stale

**Scenario**: AGENT runs orchestrator → warned → fixes implement-noted → claims done again.

**Trace**:
1. First Stop: orchestrator runs, caches verdict (60s TTL).
2. AGENT fixes implement-noted.
3. Second Stop within 60s: orchestrator sees cache fresh → silent skip → outdated verdict.

**Severity**: MEDIUM — TTL too coarse; cache should be invalidated by impl-noted mtime.

### F6 — `auto_test_runner` 600s timeout → blocks PostToolUse for 10 minutes

**Scenario**: MCP test hangs (e.g. dead PostgreSQL).

**Trace**:
1. AGENT Edits feature file.
2. PostToolUse #5 `auto_test_runner` invokes MCP test.
3. MCP hangs.
4. Hook waits 600s before timeout.
5. AGENT blocked 10 minutes for single Edit.

**Severity**: MEDIUM — config-driven mitigation possible (`auto_test.json` timeout reduce).

### F7 — `daemon_manager` kills production process by mistake

**Scenario**: `test_env.json` `pid_track_file` points at wrong PID (e.g. shared with system service).

**Trace**:
1. AGENT Edits feature-scope file.
2. `daemon_manager` reads pid → calls `Stop-Process` / `os.kill`.
3. Production service killed.

**Severity**: HIGH potential — current mitigation: PID file from start_cmd ensures isolation. But IF DEV mis-configures, no safety net.

### F8 — Hallucinated SD survives validator

**Scenario**: AGENT emits SD-N with file path that EXISTS but is unrelated to actual Edit.

**Trace**:
1. AGENT writes SD-1 "Modified `templates/agent_toolkit/README.md:5-10`" (real file, fake claim).
2. Validator: file exists ✓, line range valid ✓ → CLEAN.
3. Validator can't detect: AGENT didn't actually modify that file.

**Severity**: MEDIUM — partial mitigation via missing_sd_detector cross-check with snapshot diff (AGENT-claimed file not in actual modified-files = flag). Not implemented.

### F9 — Snapshot fills disk

**Scenario**: Long-running project with many features; auto-cleanup TTL not enforced.

**Trace**:
1. Each `/implement` creates snapshot dir.
2. TTL 7 days but no cron.
3. Over months: snapshot dirs accumulate.
4. Disk fills.

**Severity**: LOW — typical project ~5-50KB per feature, would take 1000+ features to matter.

### F10 — `evidence_audit` config drift between projects

**Scenario**: Project A has `additional_evidence_patterns` for Playwright; Project B doesn't.

**Trace**: AGENT trained on Project A's pattern → uses Playwright evidence in Project B → evidence_audit blocks because config differs.

**Severity**: LOW — handled by per-project config.

### F11 — Subprocess chain explosion in orchestrator

**Scenario**: Orchestrator launches 3 subprocess tools sequentially. Each ~2s startup.

**Trace**:
1. Orchestrator runs validator (~2s subprocess start + work).
2. Then detector (~2s + work).
3. Then annotator (~2s + work).
4. Total minimum ~6-8s overhead PER Stop event.

**Severity**: MEDIUM — slows down conversation. Mitigation: in-process import instead of subprocess.

### F12 — Spec slug mismatch with branch name

**Scenario**: Branch `feature/foo-bar` → slug derived as `foo-bar`. But spec file named `feat-foo-bar.md`.

**Trace**:
1. `spec_first_guard` looks up `<workspace>/.agent-toolkit/specs/**/foo-bar.md`.
2. Not found.
3. Warn fires every Edit.

**Severity**: LOW — DEV manual rename to match.

### F13 — Concurrent write conflicts

**Scenario**: Multiple Stop hooks write `.orchestrator_state.json` or state files concurrently.

**Trace**:
1. AGENT response triggers Stop.
2. Hooks fire sequentially per Claude Code → NO actual concurrency `[assumption]`.
3. So this is non-issue in current architecture.

**Severity**: NONE under current sequential model. Future risk if parallel hooks.

### F14 — Hook implementation crashes silently

**Scenario**: Hook script has Python error.

**Trace**:
1. Hook subprocess exits non-zero.
2. Claude Code behavior `[assumption]`: treats as silent skip OR logs but continues OR considers as block?

**Severity**: depends on Claude Code semantics. Need explicit policy.

### F15 — `implement-noted` schema drift between version

**Scenario**: v0.7.0 schema = 4 sections. v0.8.0 schema = 5 sections (hypothetical). Validator hardcoded.

**Trace**: existing impl-noted files written under v0.7.0 fail v0.8.0 validator.

**Severity**: LOW now but FUTURE consideration.

### F16 — DEV restart loop

**Scenario**: DEV restarts Cursor every time new hook ships → friction.

**Trace**:
1. v0.6.0, v0.6.2, v0.7.0, v0.7.2, v0.7.3, v0.8.0 all required restart.
2. Restarts add cumulative time cost.

**Severity**: LOW per restart, but affects xuyên suốt validation. Mitigation: batch hook changes per release.

### F17 — `acceptance-probes.json` skip-list undocumented

**Scenario**: Long-running project has 50 probes. `evidence_audit` matches against all every Stop.

**Trace**:
1. Stop fires.
2. `evidence_audit` loads 50 probes.
3. For each, checks `applies_when`.
4. Slow performance + noise.

**Severity**: LOW — performance.

## Section 5 — Fix proposals (prioritized by impact)

### P1 — Reorder Stop chain so orchestrator fires before blocking hooks

**Problem solved**: F1 (root cause of ~50% xuyên suốt drop)

**Proposal**:
- Move `implement_orchestrator.py` from position #7 → position #1 in Stop chain.
- Move `evidence_audit.py` from #1 → after orchestrator.
- Rationale: orchestrator's verdict is INFORMATION not BLOCK; should run before any blocking hook so AGENT receives audit even when blocked elsewhere.
- Alternative: orchestrator becomes non-blocking sub-event that runs in parallel (architectural change — Claude Code may not support).

**Effort**: 30 phút (settings.json reorder + test).

**Impact**: closes biggest cascade risk.

### P2 — `implement_snapshot_hook` warn when spec lacks `affected_modules`

**Problem solved**: F3 (silent grandfather → entire Layer 5 dysfunctional).

**Proposal**:
- Change current silent skip → emit stderr warn: `[snapshot-hook] grandfather: spec '{slug}' missing affected_modules; Layer 5 scope check will not engage`.
- Keep skip behavior (no block) but DEV knows.

**Effort**: 20 phút.

### P3 — `implement_orchestrator` cache invalidation by impl-noted mtime

**Problem solved**: F5 (stale cache after AGENT iter 2).

**Proposal**:
- Cache key includes impl-noted file mtime.
- Cache HIT only if mtime unchanged AND <60s old.

**Effort**: 30 phút.

### P4 — `missing_sd_detector` cross-check SD claims against actual snapshot diff

**Problem solved**: F8 (hallucinated SD survives validator).

**Proposal**:
- Detector loads snapshot diff filelist.
- For each SD-N file ref, verify file IS in modified list.
- If SD references unmodified file → flag "fabricated SD".

**Effort**: 1 ngày.

### P5 — `auto_test_runner` + `auto_run_probes` reduce default timeouts

**Problem solved**: F6 (10 minute block on single hook).

**Proposal**:
- `auto_test_runner` default 600s → 120s. Override via config.
- `auto_run_probes` default 300s → 90s.

**Effort**: 10 phút config change.

### P6 — `daemon_manager` PID safety: confirm process command-line matches expected start_cmd before kill

**Problem solved**: F7 (kill wrong process).

**Proposal**:
- Before `Stop-Process` / `os.kill`, read process command-line.
- Match against `test_env.json:process_manager.start_cmd[0]`.
- If mismatch → refuse kill + log.

**Effort**: 1-2 giờ (cross-platform proc cmdline read).

### P7 — Integration test for Stop chain ordering interactions

**Problem solved**: F1 + Layer E gap.

**Proposal**:
- `tests/test_stop_chain_interactions.py`.
- Simulate Stop with 8 envelopes representing various block scenarios.
- Assert: which hooks fire / skip based on first-block-wins semantics.
- Confirms `[assumption]` about cascade behavior.

**Effort**: 1 ngày.

### P8 — Orchestrator import tools in-process instead of subprocess

**Problem solved**: F11 (subprocess overhead).

**Proposal**:
- Replace `subprocess.run` of validator/detector/annotator with `importlib.spec_from_file_location` + direct function call.
- Save ~6s per Stop event.

**Effort**: 1 ngày.

### P9 — Document hook subprocess crash policy

**Problem solved**: F14 (silent crash semantics unclear).

**Proposal**:
- Wrap each hook's `main()` with top-level try/except.
- On exception: log to `.agent-toolkit/.hook_crash_log.json` ring buffer + exit 0.
- DEV-readable crash trail.

**Effort**: 30 phút per hook × 21 = ~2 hours total OR shared `_common.py` wrapper (30 phút).

### P10 — Schema versioning for implement-noted

**Problem solved**: F15 (future schema drift).

**Proposal**:
- Add `schema_version: 1` to implement-noted frontmatter.
- Validator handles multiple versions.

**Effort**: 1 giờ.

### P11 — Auto-cleanup snapshot TTL via post-`/verify` success hook

**Problem solved**: F9 (snapshot accumulation).

**Proposal**:
- When `/verify` passes for a slug, `verify_lint` (or new cleanup hook) calls `snapshot_cleanup(slug, force=True)`.
- Snapshot ephemeral per feature.

**Effort**: 30 phút.

### P12 — Document hook chain visualization

**Problem solved**: Layer E `[assumption]` documentation gap.

**Proposal**:
- `templates/agent_toolkit/HOOK_CHAIN.md` — diagram of fire order + block semantics + bypass markers.
- DEV reference doc.

**Effort**: 1 giờ.

### P13 — Live exercise checklist for T1 backlog

**Problem solved**: F1 verification (Layer C `[assumption]`).

**Proposal**:
- `docs/DEV_LIVE_EXERCISE.md` — step-by-step DEV manual session:
  - Restart Cursor.
  - Create branch `feature-live-test`.
  - `/plan live-test`.
  - Edit `templates/test_dummy.py`.
  - Observe stdout of each Stop hook.
  - Confirm orchestrator output appears.
- DEV runs once → empirical proof of Layer C.

**Effort**: 1 giờ doc + 1-2 giờ DEV exercise.
