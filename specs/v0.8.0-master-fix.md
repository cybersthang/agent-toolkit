# Master Fix v0.8.0 — drive xuyên suốt to 90%+ (DEV-T1 closes to 98%)

## 1. Problem

Sau 5 sprint iterative (v0.6.0 → v0.7.3), Raw Opus self-eval honest:
xuyên suốt ~66%. Root cause analysis ở `v0.8.0-master-fix-design-review.md`
identified 17 failure modes (F1-F17) + 13 fix proposals (P1-P13).

DEV target: ≥98% xuyên suốt.

## 2. Strategy

Single sprint addressing ALL Tier 1+2+3 from design review (13 fixes).
Architectural limit: AGENT alone caps ~88-90% (Layer C requires DEV
live exercise). Sprint maxes AGENT side; P13 doc + DEV exercise closes
remainder.

## 3. Affected modules

See frontmatter. Layer 5 enforce: edits outside list → block.

## 4. Implementation Decisions

D1. **Stop chain reorder (P1)**: orchestrator runs FIRST so audit
output reaches AGENT regardless of subsequent blocking hooks. Trade-off:
orchestrator may run on responses that get blocked anyway (slight
compute waste) but PRINCIPLE: information-gathering before
information-blocking.

D2. **In-process import (P8)**: replaces subprocess invocation in
orchestrator with `importlib.spec_from_file_location` + direct API call.
Saves ~6s per Stop event. Trade-off: tool module-level side effects
(stdin reads) need guards.

D3. **Cache mtime invalidation (P3)**: cache key = `(slug, impl-noted-mtime)`.
Stale cache for same slug + same mtime → use; impl-noted edit → invalidate.

D4. **Cross-check fabricated SD (P4)**: detector compares SD-N file refs
against snapshot's `modified_files` list. SD referencing unmodified file
= fabricated → flag.

D5. **Hook crash wrapper (P9)**: `_common.run_main_safe(main)` wraps
each hook's main() with try/except. On exception: log to
`.hook_crash_log.json` ring + exit 0. Migration: existing hooks opt-in
gradually (no breaking change).

D6. **Snapshot autocleanup (P11)**: `verify_lint` calls cleanup when
verify_report status='verified'. Snapshot becomes ephemeral per
feature.

D7. **PID safety (P6)**: before `Stop-Process` / `os.kill`, read
proc cmdline (cross-platform); match against `start_cmd[0]`. Mismatch
→ refuse + log.

D8. **DEV live exercise guide (P13)**: 5-step manual session covering
PreToolUse hook fire + PostToolUse + Stop + orchestrator output. Only
path to close Layer C empirically.
