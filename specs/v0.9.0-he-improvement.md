# Spec — v0.9.0 Harness Engineering improvement

## 1. Problem

Sau v0.8.1 ship, HE evaluation honest: **6.95/10 ≈ 70%**. Path β (full plan)
DEV chọn để lift ~6.95 → **~8.5** (realistic ceiling — 9.0 strict không
reachable trên Claude Code architecture).

Architectural caps:
- Dim 4 Fail-safe: ~7.5 (continuity intentional)
- Dim 6 Empirical validation: ~8.5 (no dispatcher API)

## 2. Solution

7-phase plan (B-G AGENT execute; A + H = DEV):
- Phase B: Hook consolidation 21 → ≤16
- Phase C: /hook-health dashboard + fire instrumentation
- Phase D: Warn → block config-driven migration
- Phase E: AGENT_TOOLKIT_STRICT env var dual-mode
- Phase F: In-process orchestrator imports
- Phase G: Schema enforcement + migration tools
- (Phase A DEV: P13 live exercise)
- (Phase H DEV: 3-5 features production observation)

## 3. Affected modules

See frontmatter. Layer 5 enforce: edits outside list → block.

## 4. Implementation Decisions

D1. **Sequence order — risk ascending**: G (low risk) → E → D → F → C → **B** (highest risk, last). Partial-success protected.

D2. **Phase B consolidation candidates** (specific):
- `verify_nudge.py` + `tdd_runner.py` → single `post_edit_nudges.py` (cùng warn pattern).
- `spec_drift_advisory.py` + `implement_notes_gate.py` → single `stop_warns.py`.
- `verify_lint.py` + `verify_lint_scope.py` → single `verify_gate.py` (combined coverage + scope).
- `verification_loop.py` + `verify_nudge.py` (đã merge) → review nếu redundant.
- Target: 21 → 16 (5 deletions).

D3. **Phase D `.agent-toolkit/enforce_mode.json` schema**:
```json
{
  "default": "warn",
  "per_hook": {
    "spec_first_guard": "warn",
    "implement_notes_gate": "warn",
    "verify_lint_scope": "warn"
  }
}
```

D4. **Phase E `AGENT_TOOLKIT_STRICT=1`**: flips ALL warn-mode hooks → block. CI mode. Default unset preserves dev fail-open.

D5. **Phase F in-process import** must guard against module-level side effects (stdin reads). Wrap imports in try/except.

D6. **Phase C fire instrumentation**: ring buffer 1000 events trong `.hook_fire_log.json`. CLI aggregator reads buffer.

## 5. Out-of-scope

- Phase A (DEV manual P13) — separate gating action.
- Phase H (production observation) — empirical-only.
- Layer 4 cross-feature mining.
