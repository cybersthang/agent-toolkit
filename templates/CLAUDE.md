## How Claude Code uses this workspace

1. **SessionStart**: `session_brief.py` hook injects active invariants
   and the last 3 ADRs from `.agent-toolkit/decision-log.md` into the
   first turn.
2. **UserPromptSubmit**: `intent_router.py` matches the prompt against
   skill triggers and inserts a reminder; clarification-gate questions
   MUST cite a `Searched:` line.
3. **PreToolUse(Edit|Write|MultiEdit)**: `invariant_guard.py` reads
   `.agent-toolkit/invariants.json` and **denies** any edit that strips
   a `must_keep_regex` / `must_keep_call` pattern marked `severity:
   blocker`. Bypass once with `bypass-invariant: <id>` in the next prompt.
3a. **PreToolUse(Bash)**: `git_guardrails.py` **denies** destructive git
   commands (`commit`/`push`/`add`/`--no-verify`/`--force`/`reset --hard`/
   `clean -f`/`branch -D`/`checkout .`/`restore .`). DEV-only gate per
   `feedback_no_ai_commit`. Bypass once: DEV creates
   `.agent-toolkit/.skip_git_guard_next.json` (any content; expires 600s).
4. **PostToolUse(Edit|Write|MultiEdit)**: `tdd_runner.py` nudges
   pytest on test/source edits; `verification_loop.py` nudges
   `python_syntax_check` + `python_import_check` + `xml_validate` MCP
   probes on every `.py`/`.xml` edit inside addon globs (ECC
   `verification-loop` pattern). Config:
   `.agent-toolkit/{tdd,verification}.json`.
5. **Stop**: `evidence_audit.py` blocks the response if it makes claims
   ("X is slow", "root cause is Y", "Z is missing") without a Read/Grep/
   Glob/MCP call in the same turn — agent must verify or tag `[assumption]`.
6. Auto-loads memory from `~/.claude/projects/<encoded>/memory/MEMORY.md`
   (seeded by toolkit installer; refresh with `setup.py update`).
7. Reads `AGENTS.md` and `CLAUDE.md` for project rules.
8. Spawns MCP servers from `.mcp.json` (auto-generated by installer).
9. Calls `lookup_canonical_decision` MCP tool for recurring "how do we
   do X" questions before answering.

**Opt-in quality steps (recommended, NOT auto-chained):** `/review` is a
**MANUAL opt-in** step — it is *not* auto-chained after `/implement`; run it
yourself on a module / PR / diff when you want an exhaustive review.
`/implement-notes` is **WARN/advisory by default** (`implement_notes_gate`
nudges but does not block). Both are the highest-signal steps in the flow —
running them after every `/implement` is strongly recommended. To *harden*
them into blocking gates, set `enforce_mode` to strict (copy
`.agent-toolkit/enforce_mode.strict.example.json` → `enforce_mode.json`) or
export `AGENT_TOOLKIT_STRICT=1`.

Adding a new rule: `/adr-add` (capture WHY) → `/inv-add` (translate to
enforced pattern). See `.agent-toolkit/README.md`.

Amending a **project-wide principle** (cross-spec, slow-changing):
`/constitution` — read/edit `.agent-toolkit/constitution.md`, auto-link
to a matching ADR + invariant. Use only for principles that govern HOW
the agent works, not WHAT a single feature builds.

Adding a regression test for a fixed bug: `/bug-to-test <slug>` → writes
test + registers invariant so future Edits can't strip it.

Defining acceptance evals BEFORE coding: `/eval-define <spec-slug>` —
writes pass/fail criteria into spec frontmatter; `/verify` then runs them
mechanically.

## Hard rules (Claude-specific)

- ALWAYS reply in {{RESPONSE_LANGUAGE}} unless user asks otherwise.
- ALWAYS use `{{PYTHON_BIN}}` when invoking Python — never bare `python`.
- **NEVER run `git commit` / `git push` / `git add` through the Bash tool
  unless DEV explicitly authorizes it in the current turn.** "Explicit"
  means a direct imperative ("commit cái này", "commit this", "push lên
  đi") — NOT implied by "ship feature", "fix toàn bộ", "/implement", or
  any task-completion phrase. Push is always a separate explicit step
  beyond commit. Never `--no-verify` / `--no-gpg-sign` / `--force` unless
  DEV types those flags themselves. See memory `feedback_no_ai_commit`.
- Before deep audits, read `.codex/audit_findings_locked.md` if present
  and cite the locked count verbatim.
- Save discovered preferences as memory under
  `~/.claude/projects/<encoded>/memory/`; commit portable seeds back to
  the toolkit if they should travel.
