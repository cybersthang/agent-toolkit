## How Claude Code uses this workspace

1. **SessionStart**: `session_brief.py` hook injects active invariants
   and the last 3 ADRs from `.agent-toolkit/decision-log.md` into the
   first turn.
2. **UserPromptSubmit**: `intent_router.py` matches the prompt against
   skill triggers and inserts a reminder; clarification-gate questions
   MUST cite a `Searched:` line.
3. **PreToolUse(Edit|Write|MultiEdit)**: `invariant_guard.py` reads
   `.agent-toolkit/invariants.json` and **denies** any edit that strips
   a `must_keep_regex` / `must_keep_call` pattern marked `severity:
   blocker`. Bypass once with `bypass-invariant: <id>` in the next prompt.
4. **PostToolUse(Edit|Write|MultiEdit)**: `tdd_runner.py` nudges
   pytest on test/source edits; `verification_loop.py` nudges
   `python_syntax_check` + `python_import_check` + `xml_validate` MCP
   probes on every `.py`/`.xml` edit inside addon globs (ECC
   `verification-loop` pattern). Config:
   `.agent-toolkit/{tdd,verification}.json`.
5. **Stop**: `evidence_audit.py` blocks the response if it makes claims
   ("X is slow", "root cause is Y", "Z is missing") without a Read/Grep/
   Glob/MCP call in the same turn — agent must verify or tag `[assumption]`.
6. Auto-loads memory from `~/.claude/projects/<encoded>/memory/MEMORY.md`
   (seeded by toolkit installer; refresh with `setup.py update`).
7. Reads `AGENTS.md` and `CLAUDE.md` for project rules.
8. Spawns MCP servers from `.mcp.json` (auto-generated by installer).
9. Calls `lookup_canonical_decision` MCP tool for recurring "how do we
   do X" questions before answering.

## Enforcement (mechanical, not opt-in)

| Hook | Event | Effect |
|---|---|---|
| `session_brief` | SessionStart | Brief injected — agent knows active rules from turn 1. |
| `intent_router` | UserPromptSubmit | Skill suggestions + hard-rule reminders. |
| `invariant_guard` | PreToolUse(Edit/Write/MultiEdit/NotebookEdit) | **DENY** if edit removes a must-keep pattern (severity `blocker`). Reads both `.agent-toolkit/invariants.json` and any `external_sources` declared there (e.g. project canonical registry with `enforcement` metadata). |
| `tdd_runner` | PostToolUse(Edit/Write/MultiEdit) | Nudge pytest after test/source edit (Vibe-flow Phase 3). |
| `verification_loop` | PostToolUse(Edit/Write/MultiEdit) | Nudge `python_syntax_check` + `python_import_check` + `xml_validate` MCP after edits inside addon globs. MCP prefix auto-discovered from `.mcp.json` if config is a placeholder. |
| `verify_nudge` | PostToolUse(Edit/Write/MultiEdit) | Nudge `/verify <slug>` when edit touches a file tracked by a spec with status `implementing`/`gaps-found`. 30s per-file TTL. |
| `evidence_audit` | Stop | **BLOCK** stop if claims lack Read/Grep/MCP backup → agent re-runs with verification or `[assumption]` tag. Also enforces Verify-Report probe-spread rule (ADR-007). |
| `verify_lint` | Stop | **BLOCK** stop on a Verify Report missing required sections / acceptance_eval coverage. Delegates to `.codex/lint_verify_report.py`. |
| `post_edit_verify_gate` | Stop | **BLOCK** "done" claims after Edit on a spec-tracked file without a `/verify` run this session. |
| `debug_sentry` | Stop | **BLOCK** stop when traceback/exception text appears in tool output without a fix attempt (Vibe-flow Phase 4 — toggle via `.agent-toolkit/debug.json`). |

Adding a new rule: `/adr-add` (capture WHY) → `/inv-add` (translate to
enforced pattern). See `.agent-toolkit/README.md`.

Adding a regression test for a fixed bug: `/bug-to-test <slug>` → writes
test + registers invariant so future Edits can't strip it.

Defining acceptance evals BEFORE coding: `/eval-define <spec-slug>` —
writes pass/fail criteria into spec frontmatter; `/verify` then runs them
mechanically.
