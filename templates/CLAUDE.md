## How Claude Code uses this workspace

1. **SessionStart**: `session_brief.py` hook injects active invariants
   and the last 3 ADRs from `.agent-toolkit/decision-log.md` into the
   first turn.
2. **UserPromptSubmit**: `intent_router.py` matches the prompt against
   skill triggers and inserts a reminder; clarification-gate questions
   MUST cite a `Searched:` line.
3. **PreToolUse(Edit|Write|MultiEdit)**: `invariant_guard.py` reads
   `.agent-toolkit/invariants.json` and **denies** any edit that strips
   a `must_keep_regex` / `must_keep_call` pattern marked `severity:
   blocker`. Bypass once with `bypass-invariant: <id>` in the next prompt.
4. **Stop**: `evidence_audit.py` blocks the response if it makes claims
   ("X is slow", "root cause is Y", "Z is missing") without a Read/Grep/
   Glob/MCP call in the same turn — agent must verify or tag `[assumption]`.
5. Auto-loads memory from `~/.claude/projects/<encoded>/memory/MEMORY.md`
   (seeded by toolkit installer; refresh with `setup.py update`).
6. Reads `AGENTS.md` and `CLAUDE.md` for project rules.
7. Spawns MCP servers from `.mcp.json` (auto-generated by installer).
8. Calls `lookup_canonical_decision` MCP tool for recurring "how do we
   do X" questions before answering.

## Enforcement (mechanical, not opt-in)

| Hook | Event | Effect |
|---|---|---|
| `session_brief` | SessionStart | Brief injected — agent knows active rules from turn 1. |
| `intent_router` | UserPromptSubmit | Skill suggestions + hard-rule reminders. |
| `invariant_guard` | PreToolUse(Edit/Write/MultiEdit) | **DENY** if edit removes a must-keep pattern (severity `blocker`). |
| `evidence_audit` | Stop | **BLOCK** stop if claims lack Read/Grep/MCP backup → agent re-runs with verification or `[assumption]` tag. |

Adding a new rule: `/adr-add` (capture WHY) → `/inv-add` (translate to
enforced pattern). See `.agent-toolkit/README.md`.
