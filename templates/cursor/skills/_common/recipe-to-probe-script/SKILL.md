# recipe-to-probe-script

## Purpose

Eliminate the manual "DEV writes Playwright script for this probe" step. Read the probe's `falsification.description` (free human prose), match against a public pattern library, emit a runnable script.

## Workflow

```
INPUT: probe (dict) with falsification.description.
OUTPUT: .agent-toolkit/scripts/probes/<probe_id>.py
        (Playwright sync_api script ready for falsify.py to invoke.)

For probe.falsification.description text:
  1. Tokenize the description into clauses (split on "(1)", "(2)", periods, semicolons).
  2. For each clause:
     a. Load all patterns from templates/codex/recipe_patterns/*.json.
     b. Find pattern.match_regex hits → fill template_block with capture groups.
     c. Accumulate ordered list of template_blocks.
  3. Wrap into a Playwright sync_api script skeleton (login → setup → blocks → assertions).
  4. Substitute placeholders from test_env.json (base URL, creds_ref refs).
  5. Write to .agent-toolkit/scripts/probes/<probe_id>.py.
  6. Update probe.falsification.runner.spec_file to point at the new script.

If no clauses match → emit a stub script with TODO markers + warning;
DEV completes manually.
```

## Pattern library schema

`templates/codex/recipe_patterns/<category>.json`:

```json
{
  "patterns": [
    {
      "id": "trigger-n-rpc-loop",
      "match_regex": "(?i)trigger\\s+(?P<count>\\d+)\\s+RPC",
      "imports": ["from playwright.sync_api import sync_playwright"],
      "template_block": "for i in range({count}):\n    page.evaluate(\"() => fetch('/web/dataset/call_kw', ...)\")\n",
      "vars": {"count": "from_match"}
    },
    {
      "id": "blockui-active-at-send",
      "match_regex": "(?i)blockUI\\s+active\\s+at\\s+xhr\\.send",
      "imports": [],
      "template_block": "page.evaluate(\"() => odoo.__DEBUG__.services['web.framework'].blockUI()\")\n"
    },
    {
      "id": "shadow-true-rpc",
      "match_regex": "(?i)shadow\\s*=\\s*true",
      "imports": [],
      "template_block": "page.evaluate(\"() => odoo.__DEBUG__.services['<module>.<ServiceName>']._setPendingShadowForTest(true)\")\n"
    },
    {
      "id": "sigstop-server",
      "match_regex": "(?i)SIGSTOP\\s+(?P<target>\\w+)|docker\\s+stop\\s+(?P<target2>\\w+)",
      "imports": ["import subprocess"],
      "template_block": "subprocess.run(['kill', '-STOP', str(PID)])\n"
    },
    {
      "id": "n-calls-microbench",
      "match_regex": "(?i)(?P<count>\\d+)\\s+calls\\s+.*microbench",
      "imports": ["import statistics"],
      "template_block": "samples = []\nfor i in range({count}):\n    t0 = page.evaluate('() => performance.now()')\n    page.evaluate(...)\n    samples.append(page.evaluate('() => performance.now()') - t0)\n"
    }
  ]
}
```

## Vars resolution

- `from_match`: value comes from a regex named group.
- `from_test_env`: lookup via dot-notation on test_env.json.
- `literal`: use as-is.

## Public extension

To add a new recipe phrase pattern: drop a JSON file into
`templates/codex/recipe_patterns/<your_category>.json`. Skill
auto-loads all `*.json` files alphabetically. Patterns are evaluated
in order; first match wins per clause.

Ship a unit test asserting `match_regex` matches its description
example.

## Output skeleton

```python
# AI-Code: auto-generated by recipe-to-probe-script for probe <id>.
# Source: probe.falsification.description in .agent-toolkit/acceptance-probes.json.
# Run: python <this-file>  OR  python .codex/tools/falsify.py --probe <id>
from playwright.sync_api import sync_playwright
import json, os, sys

BASE_URL = os.environ.get("HOTPOT_BASE_URL", "{test_env.url}")
LOGIN = os.environ.get("TOOLKIT_TEST_LOGIN", "{test_env.creds_ref.fallback_login}")
PASSWORD = os.environ.get("TOOLKIT_TEST_PASSWORD", "{test_env.creds_ref.fallback_password}")

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    ctx = browser.new_context()
    page = ctx.new_page()
    # ... login boilerplate ...
    # ---- recipe clauses begin ----
    {auto_generated_blocks}
    # ---- recipe clauses end ----
    # Final emit JSON evidence block:
    print("===PROBE_<id>_BEGIN===")
    print(json.dumps({"all_pass": True, "details": {...}}, ensure_ascii=False))
    print("===PROBE_<id>_END===")
    browser.close()
```

## Refuse / clarify when

- Probe lacks `falsification.description` → emit stub + warning.
- No patterns match → write TODO script + escalate DEV.
- test_env.json missing → require `/test-env-bootstrap` first.

## Linked

- `falsify.py` `playwright_python` runner consumes the generated script.
- `spec-vs-evidence-diff` warns if the recipe text grows new tokens after script generation.
